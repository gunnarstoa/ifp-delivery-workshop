O que é o Application Framework Configurator

O Application Framework Configurator é a interface única para todas as decisões de configuração do IFP. Ele apresenta 44 perguntas de configuração em três categorias: perguntas de nível superior (23), configurações de financial planning (17) e configurações de headcount (3), além de configurações adicionais. As respostas a essas perguntas determinam o que é gerado — estrutura de Módulos, Hierarquias de Listas, lógica de fórmulas e disponibilidade de método de planejamento.

O Application Framework Configurator é acessado por uma URL específica: substitua 'home' na URL do Anaplan home do seu navegador por 'applications' (não pela navegação normal do Workspace, que ainda não mostra o link Applications no v2.1.0). Por exemplo, se sua URL do Anaplan home é 'https://us1a.app.anaplan.com/home', altere a URL para 'https://us1a.app.anaplan.com/applications'. Quando o app chegar à disponibilidade geral, esse link aparecerá na navegação padrão.

i Melhor prática: gere uma cópia base primeiro

Antes de fazer qualquer alteração de configuração, gere a aplicação com todas as configurações padrão primeiro. Isso lhe dá uma cópia base — um modelo onde todas as fórmulas foram geradas corretamente, com nomes de Hierarquia padrão e Dimensões padrão intactos. Se mais tarde você tiver um erro de fórmula em sua Geração configurada, pode consultar a fórmula correta na cópia base e copiá-la. Rotule essa Geração claramente (ex.: "Base — sem alterações").

⚠ A configuração é final na Geração

Depois que você dispara a Geração, muitas decisões estruturais não podem ser alteradas sem regenerar o modelo. Níveis de Hierarquia, estrutura de entity e configuração de geografia precisam ser travados antes da Geração. Acerte essas decisões na Session Zero — são as coisas mais difíceis de mudar depois.

Visão geral do Application Framework Configurator (v2.1)

Seções de configuração

Hierarquias

A seção Hierarquias define a estrutura de Dimensões do modelo gerado. Decisões feitas aqui cascateiam por cada Módulo e não podem ser alteradas após a Geração sem uma regeneração completa.

  • Location: Quantos níveis? (ex.: Global → Region → Country → City). Quais níveis são de planejamento vs. somente roll-up?
  • Functional Area: Quais áreas funcionais estão no escopo? (Gross Profit, OpEx, Headcount, Balance Sheet). Cada área habilitada gera Módulos adicionais.
  • Entity: Quantos níveis de entity? Qual é a entity de consolidação de nível superior?
  • Product: O planejamento em nível de produto é necessário? Níveis de Hierarquia de Product?

Configurações de dimensionalidade

ℹ Impacto das perguntas de dimensionalidade no "Applies To"

Essas perguntas (13-17) impactam o "Applies To" ao gerar Módulos. O padrão tem todas as Dimensões opcionais selecionadas porque assumiu-se que todas as Dimensões serão usadas. Se alguma Dimensão for determinada como não necessária, desmarque-a.

Desmarcar Dimensões reduz a complexidade dos Módulos gerados e melhora o desempenho ao excluir combinações desnecessárias de Dimensões dos cálculos.

Nuances importantes do Configurator (aprendidas em entrega)

  • Nome de Hierarquia + pergunta do Application Framework Configurator precisam coincidir: No v2.1.0, o nome da Hierarquia na seção Hierarquias e a renomeação nas Perguntas de Configuração NÃO estão linkados. Você precisa atualizar ambos. Espera-se correção em release futura.
  • Sem espaços no final dos nomes: Espaços ao final de Hierarquias renomeadas causaram erros de Geração. Confira duas vezes se os nomes não têm espaços antes ou depois.
  • O nível folha precisa permanecer no fundo: Ao remover um nível, remova o nível do meio e mantenha a folha no fundo. Ao adicionar um nível, o sistema adiciona no fundo — arraste-o para acima do nível folha original. A folha original não pode ser renomeada com o mesmo nome do novo nível.
  • Hierarquias e Perguntas de Configuração são separadas: Existem duas perguntas por Dimensão renomeada — uma cria a Lista, outra cria os Módulos. A segunda é em minúsculas. Você precisa atualizar ambas.
  • Restore Default por Hierarquia: Cada Hierarquia tem seu próprio botão "Restore Default".
  • Ordem do Configurator vs Hierarquias: Você pode completar Hierarquias primeiro ou Perguntas de Configuração primeiro — não há ordem obrigatória.
  • Gere CapEx mesmo se não precisar agora: Módulos Polaris não usam memória significativa até conterem dados. Gere CapEx agora para evitar regeneração futura se o cliente adicionar mais tarde.
  • Dimensões personalizadas (Listas planas) são extensões, não Hierarquias: Dimensões de Lista plana (ex.: Channel) não são configuradas na seção Hierarquias — são construídas como extensões pós-geração.

Processo de Geração

Após concluir todas as Perguntas de Configuração e configurações de Hierarquia:

  1. Clique em Generate na visão geral do App Framework
  2. Insira uma descrição para esta Geração (ex.: "Base — sem alterações" ou "Customer Name — 3 níveis de entity, sem geografia")
  3. Selecione o Workspace alvo
  4. A Geração roda na nuvem — aproximadamente 20–40 minutos. Monitore o progresso na página principal /applications em vez de dentro do App Framework (o indicador de status dentro do framework pode atrasar)
  5. Quando concluído, todos os itens aparecem em verde. Itens em vermelho são erros. Itens em âmbar são avisos.
  6. Revise o Error Log. Veja a seção "Revisão do Error Log" para mais orientações.

Antes do Exercício

i Preparação

Antes de configurar o App Framework para seu cliente, identifique as perguntas-chave de escopo: níveis de entity, estrutura de geografia, áreas funcionais no escopo e quaisquer configurações especiais. Trabalhe com o cliente para esclarecer decisões ambíguas antes de começar.
