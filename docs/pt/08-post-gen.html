🔍 Verificação pós-geração — Workshop vs. realidade

Esta página cobre as verificações essenciais de alta prioridade para um modelo funcional.

Em uma implementação real de produção, a validação pós-geração é muito mais extensa — tipicamente 1 a 2 dias de revisão sistemática cobrindo:

  • Verificação de configuração (estrutura de Hierarquia, configurações de Dimensão)
  • Validação de segurança e acesso
  • Otimização de desempenho e eficiência de fórmulas
  • Documentação e preparação para handoff
  • Validação de carga de dados (integração de actuals)
  • Cenários de teste de aceitação de usuário

O checklist completo pós-geração está na Seção 5 e Apêndice A do Configuration Guide.

Este workshop foca em conceitos centrais e fluxo de trabalho. Para implementações ao vivo, sempre siga o checklist completo do Configuration Guide.

Obrigatório após cada Geração

Após a Geração ser concluída, percorra o checklist abaixo antes de fazer qualquer outro trabalho. Isso garante que você tenha uma baseline completa e estável antes de adicionar dados ou extensões.

  • Verifique se os 4 modelos existem no Workspace (Financial Planning, Headcount, Admin, CapEx)
  • Confira a contagem de objetos de aplicação — a visão geral do Application Framework mostra: modelos, UX apps, assets, objetos do data orchestrator
  • Abra os Error Logs e anote a contagem total de erros — registre isso como sua baseline
  • Revise o Error Log para verificar se todos os Módulos foram gerados corretamente — quaisquer Módulos ausentes aparecerão aqui como erros. Veja a seção Revisão do Error Log para detalhes.
  • Verifique se Line Items-chave de Módulos SYS estão populados: a Lista Planning Methods virá pré-populada. Todas as Listas de dados de produção (entity, department, accounts) virão não populadas e são populadas pelo push de dados do Data Orchestrator.
  • Confirme se as Listas de Hierarquia foram carregadas com os membros esperados. Todos os membros de Lista de Hierarquia são populados via ADO — alguns via CSV ou fonte externa conectada. Hierarquias como entity, department, IS Accounts e BS Accounts requerem uma etapa adicional de mapeamento no Admin Model para mapear membros da Lista de origem para seus equivalentes de planejamento antes de fazer push para outros modelos spoke.
  • Confira se os membros da Lista Planning Method estão populados — esse é o único item verificável neste estágio. Os Módulos SYS e ADMIN ligados aos Planning Methods são populados depois; consulte o Apêndice A3 do Configuration Guide para completar esta etapa.
  • Faça uma verificação por amostragem em um Line Item calculado em um Módulo CALC — confirme que a fórmula não está em branco
  • ▶ Assista: IFPv2 Models — walkthrough pós-geração — vídeo curto cobrindo as etapas de verificação pós-geração acima. (Abre no SharePoint · login Anaplan necessário)
⚠ Fórmulas em branco? Verifique o Error Log primeiro

A melhor forma de verificar se uma fórmula deveria estar lá mas aparece em branco é revisar o Error Log. Todos os erros de fórmula farão a fórmula ficar EM BRANCO — falham por uma referência inválida, seja por mudanças de nome de Line Item não reconhecidas ou mudanças de estrutura. Sempre verifique o Error Log antes de assumir que uma fórmula está faltando.

Mini-exercício pós-geração

1
Verifique seu modelo gerado
⏱ 20 min
Objetivo Trabalhar pelo checklist pós-geração. Identificar a contagem baseline de erros e confirmar que a estrutura do modelo está completa.
  1. 1No seu Workspace, confirme que 4 modelos estão listados. Nomeie-os em suas notas.
  2. 2Abra a página de visão geral do Application Framework. Registre: contagem de modelos, contagem de UX apps, contagem de assets, contagem de erros.
  3. 3Clique em Error Logs (canto inferior direito). Registre a contagem total de erros — esta é sua baseline.
  4. 4Abra o modelo Financial Planning. Navegue até Modules. Verifique se existe ao menos um Módulo para cada prefixo: ADMIN, SYS, DAT, INP, CALC, OUT.
  5. 5Abra um Módulo SYS. Encontre a Lista Planning Methods. Verifique se os métodos que você habilitou no Application Framework Configurator estão presentes.
  6. 6Abra um Módulo CALC (ex.: CALC OPX IS Itemized ou similar). Abra um Line Item em blueprint. Verifique se o campo de fórmula não está em branco.
  7. 7Marque seu checklist como concluído. Você está pronto para revisar os Error Logs.
