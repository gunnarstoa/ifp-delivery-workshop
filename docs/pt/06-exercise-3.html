Convenção de nomenclatura padronizada

Substitua os nomes padrão de modelo e app por um formato padronizado antes de gerar. O padrão recomendado é [ModelType]_Exercise ##_YYYYMMDD — isso garante que cada Geração produza artefatos com nomes únicos e evita erros de colisão de nome quando o mesmo exercício é executado mais de uma vez no Tenant.

Cenário

Uma empresa de utilities que opera uma usina nuclear precisa de planejamento de despesa operacional e headcount com uma work breakdown structure (WBS) detalhada para rastreamento de projetos de capital. Sem balance sheet, cashflow ou margin planning.

Configuração do modelo

  • Apenas Operating Expense e Headcount
  • NÃO usará: top-down targets, margin planning, CapEx, balance sheet, cashflow ou alocações
  • Configurará extensão para rastreamento de projetos CapEx & POs
  • Strata de tempo: mensal
  • Tempo de Geração: a definir

Principais decisões de configuração

  • Functional Area repropositada como WBS (Work Breakdown Structure) com 5 níveis para Hierarquia detalhada de projetos
  • Várias Dimensões personalizadas para rastreamento de projetos e compras (tratadas na extensão)
  • Estrutura rica de atributos no Cost Center para análise multidimensional
  • Escopo mínimo: OpEx + Headcount apenas, mantendo a configuração simples
⚠ Dimensões personalizadas são extensões

PO Headers, PO Items e outras Listas personalizadas para este caso de uso NÃO fazem parte da Geração base. Serão adicionadas como parte da fase de extensão, após o modelo ser gerado com as Dimensões core configuradas.
