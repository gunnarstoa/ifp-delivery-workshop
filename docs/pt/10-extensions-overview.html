O que são extensões

Extensões são adições ou modificações específicas do cliente ao modelo IFP base que o Application Framework Configurator não consegue produzir. Elas se enquadram em três categorias — e a categoria determina o risco e a abordagem.

Tipo de extensãoO que éRisco de upgradeQuando usar
ConfigurarUse o Application Framework Configurator para habilitar uma capacidade nativaNenhum — isto é suportadoSempre tente isto primeiro. Se o Application Framework Configurator tem uma configuração para isso, use o Application Framework Configurator.
Estender / ConectarAdicione novos Line Items, Módulos ou Listas sem tocar nas fórmulas do modelo baseBaixo — itens novos não são sobrescritos por upgradesQuando o cliente precisa de algo que o Application Framework Configurator não cobre, mas você pode adicionar sem mudar a lógica do modelo base.
Alterar / Modificar / ExcluirAlterar uma fórmula, Lista ou Módulo existente do modelo baseAlto — alterações podem ser perdidas ou quebradas no upgradeÚltimo recurso. Documente todo override com um Data Tag. Mantenha documentação externa.

7 melhores práticas para extensões IFP

1

Configure primeiro

A configuração via Application Framework é o caminho primário para definir a estrutura de uma aplicação com base nas entradas e requisitos do cliente.

2

Estenda ou conecte em seguida

Extensões devem sempre ser aditivas, ou seja, adicionar nova capacidade sem alterar a lógica existente.

3

Modifique com cuidado

Modificar objetos existentes — especialmente aqueles que não podem ser configurados — pode impactar tanto a configurabilidade quanto a upgradeability.

4

Remova com cautela

Remover componentes de uma aplicação pós-configurada pode criar esforço adicional em reconfigurações futuras ou ao adotar uma nova versão da aplicação.

5

Mantenha uma convenção de nomenclatura consistente

Nomenclatura consistente torna extensões mais fáceis de manter, suportar e fazer upgrade.

6

Agrupe extensões relacionadas

Agrupar várias extensões torna-as mais fáceis de identificar e gerenciar, especialmente durante upgrades.

7

Documente todas as extensões

Todas as extensões e modificações devem ser documentadas claramente.

Data Tags — crie-as primeiro

⚠ Marque antes de construir

Antes de fazer quaisquer alterações de extensão em um modelo IFP, crie dois Data Tags: Extension (para todos os novos Line Items que você adicionar) e Formula Override (para qualquer fórmula do modelo base que você alterar). Essas tags são o único registro dentro do modelo do que você alterou. Sem elas, um futuro consultor não tem como saber.

⚠ Formula Overrides e upgrades

Formula overrides em Line Items do modelo base são a ação de extensão de maior risco. Quando a Anaplan libera um update do IFP, formula overrides podem ser perdidos ou quebrados. Sempre marque-os com o Data Tag Formula Override e documente externamente. Após qualquer upgrade do IFP, filtre pelo Data Tag Formula Override e revalide cada um.
