Extension Exercise 1: Manual Adjustments

Add 2 new manual adjustment line-items to the OpEx module and modify the aggregation formula

Extensions30 minGuided Practice

Business Requirement

Need 2 additional types of manual adjustments for expense inputs.

Solution

  • Add new line-items to the affected module
  • Modify the aggregation formula to include the new line-items

Steps

1

Open the Expense Planning Module

Open module INP OPX Expense Planning in the "Financial Planning" model.

2

Add 2 New Line-Items

Add 2 new line-items in the "Direct Input Method" section of the blueprint:

  • Manual Adjustment #1
  • Manual Adjustment #2
  • Set the Format of the new line-items (same as "Amount")
  • Set the Write Access Driver for each new line-item
  • Document what the line-items are for in the "Notes" field
3

Modify the Aggregation Formula

We need to modify the formula for line-item Direct Input Fcst to include the 2 new line-items:

  • Modify the formula
  • Document what you've done:
    • Use the "Notes" field of the module blueprint
    • Keep a master list of overrides in an offline document, including the original and updated formula syntax
4

Update Line-Item Subset and Admin Settings

Add the 2 new line-items to the LISS (Line-Item Subset for Show/Hide):

  • Open line-item subset "LISS: OpEx Filter"
  • Check the Boolean for the 2 new adjustments
  • Open module "ADMIN LISS OpEx Filter" and set the "Forecast Method" to "Direct Input" for each of the 2 new line-items
5

Verify in the UX App

Open page "OpEx – planning" in the UX App:

  • Select account "6005 Office Equipment"
  • Check that the logic is working properly
