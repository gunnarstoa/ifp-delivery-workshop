Extension Exercise 1: Manual Adjustments

Add two manual-adjustment line items to the OpEx module, wire them into the aggregation, and verify in the UX

Extensions⏱ 30 minGuided practice

What you'll do

  • Add two new manual-adjustment line items to the INP OPX Expense Planning module
  • Modify the Direct Input Fcst aggregation formula to include the new line items
  • Register the new line items in the OpEx Line-Item Subset (LISS) and the ADMIN LISS module so they appear in the UX
  • Verify the end-to-end flow in the UX with a real account (6005 Office Equipment)

The scenario

The client needs two additional types of manual adjustments for expense inputs — values the planner enters directly that flow into the Direct Input forecast alongside the existing inputs. You'll add the line items, wire them into the aggregation, and make sure they show up in the right UX page.

Before you start: how extensions stay safe

⚠ Rules to follow throughout the exercise
  • Add — don't replace. Generated modules and line items stay as they are. Extensions append new line items and adjust formulas; they don't rewrite the generated structure.
  • Use the module Notes field. Document why each new line item exists and what the aggregation formula now sums. Future-you (and the next Anaplan upgrade) will thank you.
  • Keep an offline change log. When an Anaplan IFP release lands, every extension has to be re-applied to the freshly generated model. A master list of overrides — original and updated formula syntax — makes that re-application straightforward instead of detective work.
  • Update the LISS, or the UX won't see it. A line item that isn't registered in its Line-Item Subset doesn't appear on the relevant UX page, even if the formula works perfectly.

Step 1 — Open the Expense Planning module

  1. Open the Financial Planning model from your workspace.
  2. Navigate to Modules and open INP OPX Expense Planning.
  3. Switch to Blueprint view so you can see line items, formats, formulas, and the Notes field.

Step 2 — Add the two new line items

You're adding both line items in the Direct Input Method section so they live alongside the inputs the planner already enters directly.

  1. In the Direct Input Method section of the blueprint, create a new line item named Manual Adjustment #1.
  2. Set its Format to match the existing Amount line item (so it aggregates the same way).
  3. Set the Write Access Driver the same as Amount so the same users who can enter Amount can enter the adjustment.
  4. Use the Notes field to document what this adjustment is for.
  5. Create a second line item named Manual Adjustment #2 and repeat the format, write-access, and notes steps.

Step 3 — Wire the new line items into the aggregation formula

The line item Direct Input Fcst rolls up the Direct Input section. You'll add the two new adjustments to its formula and document the change.

  1. Open the formula for Direct Input Fcst.
  2. Add references to Manual Adjustment #1 and Manual Adjustment #2 so they aggregate into the Direct Input forecast alongside Amount.
  3. Document the change in the module's Notes field: which line items were added, and the before/after of the Direct Input Fcst formula.
  4. Add the same entry to your offline change log so the next upgrade can re-apply this extension without you having to re-derive it.

Step 4 — Register the new line items in the LISS

A Line-Item Subset (LISS) controls which line items the UX shows on a page. If you skip this step, the line items exist in the module and aggregate correctly — but the UX won't display them.

  1. Open the line-item subset LISS: OpEx Filter.
  2. Check the Boolean for Manual Adjustment #1 and Manual Adjustment #2.
  3. Open the module ADMIN LISS OpEx Filter.
  4. For each new adjustment, set the Forecast Method to Direct Input — that tells the UX which method group these line items belong to.

Step 5 — Verify the end-to-end flow in the UX

The whole point of this extension is that a planner can enter a manual adjustment and see it flow into the forecast. Don't sign it off until you've watched that happen with a real account.

  1. Open the UX app and navigate to the OpEx – planning page.
  2. Select account 6005 Office Equipment.
  3. Confirm the two new manual-adjustment line items appear in the Direct Input section.
  4. Enter a small value in Manual Adjustment #1. Confirm the value flows into Direct Input Fcst.
  5. Repeat with Manual Adjustment #2 and confirm it lands in the same forecast.
  6. If either adjustment doesn't show up: check the LISS Boolean and the ADMIN LISS Forecast Method. If the value shows up but doesn't roll into Direct Input Fcst: re-read the formula and compare line-item names exactly.
End-of-step screenshot to capture OpEx – planning UX page for account 6005 Office Equipment showing both Manual Adjustment #1 and #2 in the Direct Input section, with sample values entered and visibly rolling into Direct Input Fcst.

Once both manual adjustments enter values that flow into Direct Input Fcst in the UX, run the check below.

Manual adjustments extension wired up.

  • Line items added → Manual Adjustment #1 and #2 in INP OPX Expense Planning
  • Aggregation → both feed into Direct Input Fcst
  • LISS → both registered in LISS: OpEx Filter and ADMIN LISS OpEx Filter (Forecast Method = Direct Input)
  • UX verification → both visible and rolling into the forecast on the OpEx – planning page for account 6005 Office Equipment
  • Documentation → recorded in module Notes and the offline change log

Extension is complete and verified end-to-end. Move on to Exercise 2.

Quick check

Correct. The LISS controls which line items the UX exposes on a page. A line item that exists in the module and aggregates correctly is still invisible to the UX until it's checked in LISS: OpEx Filter and assigned a Forecast Method in ADMIN LISS OpEx Filter.

Correct. Notes give the next person reading the module the "why" inline. The offline log is your re-application script — when Anaplan ships a new IFP base, the freshly generated model has the original Direct Input Fcst formula again, and your log is what tells you exactly which adjustments to add back.
