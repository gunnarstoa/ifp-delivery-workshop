Extension Exercise 2: Channel Revenue Planning

Add a new flat list (Channel) and revenue planning module integrated into Gross Profit Planning

Extensions60 minGuided Lab

Business Requirement

Need 1 type of revenue planning, for example by Channel.

Solution

  • Add new flat list of Channel with top level – All Channels
  • Create a new module with the existing revenue planning dimensions plus additional dimension of Channel
  • Add new line-items to the affected module
  • Modify the aggregation formula to include the new line-items

Steps

Step 1: Create the Channel List

  1. In the Data Model editor, create a new List: EXT Channel
    • Naming convention: "EXT" prefix indicates this is an extension (not auto-generated)
    • Flat list (no hierarchy) — just discrete channel names
  2. Add the Top Level item: "All Channels"
    • This aggregates all individual channel values
  3. Add 3 sample channel items:
    • Channel 1
    • Channel 2
    • Channel 3
  4. Save the list
ℹ Naming Convention: EXT Prefix

Lists and modules created as extensions are prefixed with "EXT" to distinguish them from auto-generated content. This makes it easy to track what was added post-generation during upgrades.

Step 2: Create the Channel Revenue Planning Module

  1. Create a new module: EXT INP Channel Revenue Planning
  2. Define the module dimensions:
    • Time (standard dimension)
    • All dimensions from INP Gross Profit Planning (Product, Job, Entity, etc.)
    • Channel (your new dimension — EXT Channel list)
  3. Add line-items to the module:
    • Price — unit price per channel
    • Volume — units sold per channel
    • Total — calculated revenue (Price × Volume)
  4. Configure the "Total" line-item with formula: Price × Volume
  5. Save the module

Step 3: Add a New Planning Method

  1. Open the "SYS by Planning Methods" module
  2. In the "List Planning Methods", add a new item: "Channel Planning"
  3. Save the change
⚠ Planning Methods Must Be Registered

Every new planning approach (Channel Planning, Direct Input, etc.) must be registered in the Planning Methods list. Without this registration, the UX won't recognize it as a valid option for users.

Step 4: Configure Planning Method Availability

  1. Open module "ADMIN Define Product to Planning Method"
  2. For "Service Offering 1" (or your target product), add "Channel Planning" to the list of available planning methods
  3. This tells the system: "Service Offering 1 can be planned using Channel Planning"
  4. Save the changes

Step 5: Update the Gross Profit Planning Module

Now integrate Channel Planning into the main planning flow by adding reference line-items to INP Gross Profit Planning:

  1. Open module "INP Gross Profit Planning"
  2. Add 3 new line-items before the "Final Amount" line-item:
    • Channel Planning — heading/section indicator
      • Set as Heading 1 (styling/organization only, no data)
    • Use Channel Planning Method? — Boolean flag
      • Format: Boolean
      • Purpose: User selects "Yes" to enable channel-level planning
      • Copy settings from similar line-item (e.g., "Use COGS Pct of Revenue Method?")
    • Channel Planning Revenue — the aggregated result
      • Format: Same as "Final Amount"
      • This pulls data from EXT INP Channel Revenue Planning and sums by channel

Step 6: Create the Channel Planning Revenue Formula

  1. For the "Channel Planning Revenue" line-item, create a formula that:
    • Links to EXT INP Channel Revenue Planning, line-item "Total"
    • Sums across the Channel dimension (aggregates all channels)
    • Only calculates when "Use Channel Planning Method?" = True
    • Returns 0 or blank when the method is not selected
  2. Formula pattern (example):
    • IF(Use Channel Planning Method?, SUM(EXT INP Channel Revenue Planning.Total), 0)
  3. Adjust the exact formula based on your model's naming and structure
  4. Document the formula in the module blueprint Notes field

Step 7: Update Line-Item Subsets (LISS)

Register your new line-items in the appropriate subsets so they appear in the right UX pages:

  1. Open "LISS: Margin Filter"
  2. Check the Boolean for your 3 new line-items:
    • Channel Planning (heading)
    • Use Channel Planning Method?
    • Channel Planning Revenue
  3. Save the changes

Step 8: Update Aggregation Formulas

Update the formulas in existing line-items to include Channel Planning Revenue:

  1. In module "INP Gross Profit Planning", open the formula for "Final Amount"
  2. Update the formula to include:
    • + Channel Planning Revenue (in addition to existing components)
  3. Update the formula for "SUM: Revenue" if it exists:
    • Add + Channel Planning Revenue to ensure channel-based revenue is included in summaries
  4. Document both formula changes in your offline change log
ℹ Formula Update Strategy

When updating existing formulas to reference new extensions, use conditional logic:

  • IF(Use Channel Planning Method?, Channel Planning Revenue, 0)

This prevents errors when the method isn't selected and makes the logic clear to future maintainers.

Step 9: Validate in the UX

  1. Open the UX App
  2. Navigate to the Revenue & COGS - planning page for a sample product (Service Offering 1)
  3. Verify:
    • The "Use Channel Planning Method?" Boolean appears and is editable
    • When you set it to "Yes", the planning interface adjusts
    • You can navigate to the Channel Revenue Planning page and enter Price and Volume
    • The totals flow back to Revenue & COGS - planning correctly
    • All summary calculations are accurate
  4. If something is missing or broken:
    • Check that all line-items are added to the LISS
    • Verify the formula syntax matches your model's naming conventions
    • Compare against a working reference module

Key Takeaways

  • Extensions preserve generated structure — you're adding, not replacing
  • Planning Methods organize user choices — always register new planning approaches
  • Line-Item Subsets control visibility — new content only appears if added to LISS
  • Formulas must be conditional — only calculate what the user selected
  • Documentation is critical — every change must be logged for future upgrades
  • Test end-to-end — from data entry through final summary calculations
⚠ Upgrade Consideration

When Anaplan releases a new IFP version, all extensions you've built must be re-applied to the newly generated model. Keeping detailed documentation (what you changed, why, original vs. new formulas) makes this process much faster and less error-prone.
