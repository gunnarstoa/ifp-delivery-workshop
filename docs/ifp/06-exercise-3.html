Configuration Exercise 3: Nuclear Plant – OpEx and Headcount

Hands-on exercise to configure IFP for a utilities company

Configuration30 minGuided Practice
⚠ Reset Before You Start — Each Exercise is Net-New

This exercise is independent of the previous one. Before you begin, restore the Application Framework Configurator to its default state so prior answers don't carry over.

Important: Hierarchies must be restored to default one at a time — there is no global "restore all hierarchies" button. Work through each hierarchy individually.

Complete all three resets in this order:

  1. Restore Default for Configuration Questions — In the Configuration Questions section, click Restore default. All gray "CUSTOM ANSWER" indicators should flip back to "DEFAULT ANSWER".
  2. Restore Default for each Hierarchy — In the Hierarchies section, open each hierarchy (Entity, Account, Location, etc.) and click Restore default for each one individually. Confirm levels return to the documented defaults.
  3. Clear Dataspace Objects — Remove any objects from your assigned Dataspace that were created during the prior exercise so generation starts from a clean slate.

Standardized Naming Convention

Replace the default model and app names with a standardized format before generating. The recommended pattern is [ModelType]_Exercise ##_YYYYMMDD — this ensures every generation produces uniquely-named artifacts and avoids name-collision generation errors when the same exercise is run more than once in the tenant.

Scenario

A utilities company operating a nuclear facility needs operating expense and headcount planning with a detailed work breakdown structure (WBS) for capital project tracking. No balance sheet, cashflow, or margin planning required.

Model Setup

  • Operating Expense and Headcount only
  • Will NOT use: top-down targets, margin planning, CapEx, balance sheet, cashflow, or allocations
  • Will configure extension for CapEx projects & PO tracking
  • Time strata: Monthly
  • Estimated generation time: 20–40 minutes

Key Configuration Decisions

  • Functional Area repurposed as WBS (Work Breakdown Structure) with 5 levels for detailed project hierarchy
  • Multiple custom dimensions for project and procurement tracking (handled in extension)
  • Rich attribute structure on Cost Center for multi-dimensional analysis
  • Minimal scope: OpEx + Headcount only, keeping configuration simple
⚠ Custom Dimensions Are Extensions

PO Headers, PO Items, and other custom lists for this use case are NOT part of the base generation. They will be added as part of the extension phase, after the model is generated with the core dimensions configured.
