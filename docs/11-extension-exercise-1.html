Extension Exercise 1: Manual Adjustments

Add custom expense adjustment line items to a generated model

Day 245 minGuided Lab
📖 Tell Business Requirement Why partners need to add manual adjustment line items post-generation.
👁 Show Step-by-Step Walk-Through Facilitator demonstrates the exact configuration and formula changes required.
✅ Do Configure Your Own Participants replicate the changes in their own model and validate the logic.
💡 Key Principle: Extensions Require Pre-Work

Complete all work in the Configuration Guide prior to starting any extensions and/or overrides. Model structure must be locked before adding custom elements.

Business Scenario

Business Requirement: The finance team needs to track two additional types of manual adjustments for expense inputs beyond the default structure. Examples might include:

  • Mid-year budget adjustments
  • Special project cost allocations
  • One-time extraordinary items

Solution Approach: Add new line-items to the OpEx Expense Planning module, modify the aggregation formula to include them, and configure write access and UX visibility.

Step-by-Step Implementation

Step 1: Add New Line-Items to the Module

  1. Open module INP OPX Expense Planning in the "Financial Planning" model
  2. Navigate to the "Direct Input Method" section of the blueprint
  3. Add 2 new line-items:
    • Manual Adjustment #1
    • Manual Adjustment #2
  4. Set the Format of the new line-items to match the existing "Amount" line-item format
  5. Set the Write Access Driver for each new line-item (e.g., allow users to write directly or via a specific planning method)
  6. Document what the line-items are for in the module blueprint "Notes" field
⚠ Documentation Is Critical

Every extension you make must be documented. Include:

  • What the new line-item is for (business use case)
  • Which teams will have write access
  • Any formula dependencies
  • Dates when the extension was added

This becomes essential during upgrades and troubleshooting.

Step 2: Modify the Aggregation Formula

The existing formula for Direct Input Fcst currently sums only the pre-generated line-items. You need to update it to include your new adjustments.

  1. Open the formula for line-item "Direct Input Fcst" in the blueprint
  2. Modify the formula to include the 2 new manual adjustment line-items:
    • Add + Manual Adjustment #1 + Manual Adjustment #2 to the SUM statement
    • Keep the formula syntax consistent with existing patterns
  3. Document what you changed in the "Notes" field of the module blueprint
  4. Maintain a master list of overrides in an offline document (Excel, Notion, etc.) with:
    • Original formula syntax
    • Updated formula syntax
    • Date of change
    • Reason for change

Step 3: Update the Line-Item Subset (LISS)

The LISS (Line Item Subset) controls which line-items appear in specific UX modules. You need to register your new line-items.

  1. Open line-item subset "LISS: OpEx Filter"
  2. Check the Boolean checkbox for both new manual adjustment line-items
  3. Save the changes

Step 4: Configure Planning Method Access

Determine how users will interact with these adjustments in the UX:

  1. Open module "ADMIN LISS OpEx Filter"
  2. For each of the 2 new line-items, set the "Forecast Method" to "Direct Input"
  3. This allows users to manually enter adjustment values in the planning interface
  4. Save and verify the settings

Step 5: Validate the Logic in the UX

Test your changes end-to-end:

  1. In the UX App, open page "OpEx – Planning"
  2. Select account "6005 Office Equipment" (or any account to test)
  3. Verify the new adjustment line-items appear and are editable
  4. Enter a test value in "Manual Adjustment #1"
  5. Confirm that:
    • The value is captured
    • The formula includes the adjustment in the "Direct Input Fcst" total
    • The logic cascades correctly to upper-level summaries
  6. If something doesn't work: Compare your formula to the reference (base copy) and correct any syntax errors
ℹ Troubleshooting Tip

If your new adjustment line-items don't appear in the UX or the formula doesn't calculate, check:

  • Did you add the line-items to the LISS?
  • Did you update the aggregation formula correctly?
  • Did you set the Planning Method to "Direct Input"?
  • Compare your formula syntax character-by-character against a working example

Key Takeaways

  • Extensions follow a consistent pattern: Add → Configure → Update Formula → Validate
  • Documentation is not optional — it's a prerequisite for upgrades and collaboration
  • Always test in the UX before declaring the extension complete
  • Keep a master change log so you know exactly what you modified and why
  • Reference implementations (base copy) are your best debugging tool
