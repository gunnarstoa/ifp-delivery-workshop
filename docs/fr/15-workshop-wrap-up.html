Checklist de nettoyage du Tenant

Le Tenant de formation est partagé entre de nombreuses promotions. Suivre ces cinq étapes avant de partir permet de garder le Tenant dans un état propre pour le prochain groupe de participants — et de maintenir ces ateliers viables dans la durée.

⚠ Ne supprimez pas les modèles ou applications TRAIN

Les modèles et applications TRAIN pré-chargés (ceux préfixés par TRAIN, par exemple IFPv2.0.0-train-base-financial-planning) appartiennent à tous les participants et doivent rester en place. Supprimez uniquement les apps, modèles et objets que vous avez créés pendant les exercices.

  • Supprimez les applications que vous avez configurées pendant les exercices — celles sans le préfixe TRAIN
  • Supprimez les modèles générés à partir de vos applications configurées — ceux sans le préfixe TRAIN
  • Supprimez les objets que vous avez ajoutés à votre Dataspace attribué pendant les exercices
  • Réinitialisez les hiérarchies et les questions de configuration aux valeurs par défaut dans l'Application Framework Configurator (une hiérarchie à la fois — il n'y a pas de réinitialisation globale)
  • Restaurez l'ID pré-extension du modèle Financial Planning
    • Pour les Tenants Application Training 00 à 60, l'ID pré-extension est 2064565
    • Pour les Tenants utilisés dans les ateliers EMEA, l'ID peut différer — confirmez avec l'administrateur du Tenant EMEA avant de restaurer
