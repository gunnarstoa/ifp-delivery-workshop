Exercice d'extension 2 : Channel Revenue Planning

Ajoutez une nouvelle Liste plate (Channel) et un Module de planification du revenu intégré à Gross Profit Planning

Extensions60 minExercice guidé
🌐 Avis de traduction

Ceci est une traduction automatique de la version originale en anglais. Pour la version officielle et la plus à jour, consultez la version anglaise. Signalez tout problème de traduction à votre contact Anaplan.

📖 Présenter Exigence métier Le client a besoin d'un type de planification du revenu selon une nouvelle Dimension (par exemple, Channel).
👁 Montrer Visite guidée étape par étape L'animateur parcourt la création des Listes, des Modules, des méthodes de planification et l'intégration aux flux de revenu existants.
✅ Faire Construire l'extension Exécutez le workflow en 7 étapes pour ajouter la planification du revenu par channel de bout en bout.

Exigence métier

Besoin d'un type de planification du revenu, par exemple par Channel.

Solution

  • Ajouter une nouvelle Liste plate Channel avec un niveau supérieur – All Channels
  • Créer un nouveau Module avec les Dimensions existantes de planification du revenu plus une Dimension supplémentaire : Channel
  • Ajouter de nouveaux Line Items au Module concerné
  • Modifier la formule d'agrégation pour inclure les nouveaux Line Items

Étapes

Étape 1 : Créer la Liste Channel

  1. Dans l'éditeur Data Model, créez une nouvelle Liste : EXT Channel
    • Convention de nommage : le préfixe « EXT » indique qu'il s'agit d'une extension (non auto-générée)
    • Liste plate (sans Hiérarchie) — juste des noms de channels discrets
  2. Ajoutez l'élément Top Level : « All Channels »
    • Cela agrège les valeurs de tous les channels individuels
  3. Ajoutez 3 channels d'exemple :
    • Channel 1
    • Channel 2
    • Channel 3
  4. Enregistrez la Liste
ℹ Convention de nommage : préfixe EXT

Les Listes et Modules créés en tant qu'extensions sont préfixés par « EXT » pour les distinguer du contenu auto-généré. Cela permet de suivre facilement ce qui a été ajouté après la génération lors des mises à niveau.

Étape 2 : Créer le Module Channel Revenue Planning

  1. Créez un nouveau Module : EXT INP Channel Revenue Planning
  2. Définissez les Dimensions du Module :
    • Time (Dimension standard)
    • Toutes les Dimensions de INP Gross Profit Planning (Product, Job, Entity, etc.)
    • Channel (votre nouvelle Dimension — Liste EXT Channel)
  3. Ajoutez les Line Items au Module :
    • Price — prix unitaire par channel
    • Volume — unités vendues par channel
    • Total — revenu calculé (Price × Volume)
  4. Configurez le Line Item « Total » avec la formule : Price × Volume
  5. Enregistrez le Module

Étape 3 : Ajouter une nouvelle Planning Method

  1. Ouvrez le Module « SYS by Planning Methods »
  2. Dans la « List Planning Methods », ajoutez un nouvel élément : « Channel Planning »
  3. Enregistrez la modification
⚠ Les Planning Methods doivent être enregistrées

Chaque nouvelle approche de planification (Channel Planning, Direct Input, etc.) doit être enregistrée dans la Liste Planning Methods. Sans cet enregistrement, l'UX ne la reconnaîtra pas comme une option valide pour les utilisateurs.

Étape 4 : Configurer la disponibilité de la Planning Method

  1. Ouvrez le Module « ADMIN Define Product to Planning Method »
  2. Pour « Service Offering 1 » (ou votre produit cible), ajoutez « Channel Planning » à la liste des Planning Methods disponibles
  3. Cela indique au système : « Service Offering 1 peut être planifié en utilisant Channel Planning »
  4. Enregistrez les modifications

Étape 5 : Mettre à jour le Module Gross Profit Planning

Intégrez maintenant Channel Planning dans le flux de planification principal en ajoutant des Line Items de référence à INP Gross Profit Planning :

  1. Ouvrez le Module « INP Gross Profit Planning »
  2. Ajoutez 3 nouveaux Line Items avant le Line Item « Final Amount » :
    • Channel Planning — indicateur de section/en-tête
      • Définissez-le comme Heading 1 (mise en forme/organisation uniquement, pas de données)
    • Use Channel Planning Method? — drapeau Booléen
      • Format : Boolean
      • Objectif : l'utilisateur sélectionne « Oui » pour activer la planification au niveau channel
      • Copiez les paramètres d'un Line Item similaire (par exemple, « Use COGS Pct of Revenue Method? »)
    • Channel Planning Revenue — le résultat agrégé
      • Format : identique à « Final Amount »
      • Cela récupère les données de EXT INP Channel Revenue Planning et les somme par channel

Étape 6 : Créer la formule Channel Planning Revenue

  1. Pour le Line Item « Channel Planning Revenue », créez une formule qui :
    • Renvoie vers EXT INP Channel Revenue Planning, Line Item « Total »
    • Somme sur la Dimension Channel (agrège tous les channels)
    • Calcule uniquement lorsque « Use Channel Planning Method? » = True
    • Retourne 0 ou vide lorsque la méthode n'est pas sélectionnée
  2. Modèle de formule (exemple) :
    • IF(Use Channel Planning Method?, SUM(EXT INP Channel Revenue Planning.Total), 0)
  3. Ajustez la formule exacte en fonction du nommage et de la structure de votre modèle
  4. Documentez la formule dans le champ Notes du blueprint du Module

Étape 7 : Mettre à jour les Line Item Subsets (LISS)

Enregistrez vos nouveaux Line Items dans les subsets appropriés afin qu'ils apparaissent dans les bonnes pages UX :

  1. Ouvrez « LISS: Margin Filter »
  2. Cochez le Booléen pour vos 3 nouveaux Line Items :
    • Channel Planning (en-tête)
    • Use Channel Planning Method?
    • Channel Planning Revenue
  3. Enregistrez les modifications

Étape 8 : Mettre à jour les formules d'agrégation

Mettez à jour les formules des Line Items existants pour inclure Channel Planning Revenue :

  1. Dans le Module « INP Gross Profit Planning », ouvrez la formule de « Final Amount »
  2. Mettez à jour la formule pour inclure :
    • + Channel Planning Revenue (en plus des composants existants)
  3. Mettez à jour la formule de « SUM: Revenue » si elle existe :
    • Ajoutez + Channel Planning Revenue pour garantir que le revenu par channel est inclus dans les récapitulatifs
  4. Documentez les deux modifications de formule dans votre journal de changements hors-ligne
ℹ Stratégie de mise à jour de formule

Lors de la mise à jour de formules existantes pour référencer de nouvelles extensions, utilisez une logique conditionnelle :

  • IF(Use Channel Planning Method?, Channel Planning Revenue, 0)

Cela évite les erreurs lorsque la méthode n'est pas sélectionnée et rend la logique claire pour les futurs mainteneurs.

Étape 9 : Valider dans l'UX

  1. Ouvrez l'UX App
  2. Naviguez vers la page Gross Profit Planning pour un produit d'exemple (Service Offering 1)
  3. Vérifiez :
    • Le Booléen « Use Channel Planning Method? » apparaît et est modifiable
    • Quand vous le réglez sur « Oui », l'interface de planification s'adapte
    • Vous pouvez naviguer vers la page Channel Revenue Planning et saisir Price et Volume
    • Les totaux remontent correctement vers Gross Profit Planning
    • Tous les calculs récapitulatifs sont exacts
  4. Si quelque chose manque ou est cassé :
    • Vérifiez que tous les Line Items sont ajoutés au LISS
    • Vérifiez que la syntaxe de la formule correspond aux conventions de nommage de votre modèle
    • Comparez avec un Module de référence qui fonctionne

Points clés à retenir

  • Les extensions préservent la structure générée — vous ajoutez, vous ne remplacez pas
  • Les Planning Methods organisent les choix utilisateurs — enregistrez toujours les nouvelles approches de planification
  • Les Line Item Subsets contrôlent la visibilité — le nouveau contenu n'apparaît que s'il est ajouté au LISS
  • Les formules doivent être conditionnelles — ne calculez que ce que l'utilisateur a sélectionné
  • La documentation est critique — chaque changement doit être enregistré pour les futures mises à niveau
  • Testez de bout en bout — de la saisie des données jusqu'aux calculs récapitulatifs finaux

⚠ Considération de mise à niveau

Lorsqu'Anaplan publie une nouvelle version d'IFP, toutes les extensions que vous avez construites doivent être ré-appliquées au modèle nouvellement généré. Tenir une documentation détaillée (ce que vous avez modifié, pourquoi, formules originales vs nouvelles) rend ce processus beaucoup plus rapide et moins sujet aux erreurs.
