Lab C: Headcount × Rate Planning Method

Build a headcount-driven rate planning method end-to-end — the most detailed lab

Lab C60 minHands-On
📖 Tell The Business Requirement Plan Training expenses as headcount × average rate + adjustment.
👁 Show Complete 5-Step Build Facilitator walks through the complete build live using the micro-lesson video flow.
✅ Do Build End to End Participants build the Headcount × Rate method following the seven mini-labs.

Business Requirement

For the Training GL account, FintechCo wants to plan expenses using headcount × an average rate per person, plus an adjustment. The formula is:

Headcount × Rate + Adjustment = Total

In actuals periods: actuals headcount × (actuals GL balance ÷ actuals headcount) = actual amount.
In forecast periods: headcount from the Headcount model × user-input rate + user-input adjustment = forecast total.

This is the most detailed lab — seven mini-labs following the exact flow from the Day 2 micro-lesson video.

ℹ Model to Use

Work in the IFPv2.0.0 train-base-financial-planning model (or what your facilitator specifies). This lab builds on the same model you used for Lab B.

C1
Create Planning Method in List
⏱ 5 min
Objective Register the new Headcount × Rate planning method in the Planning Methods list before configuring its behavior.
  1. 1Navigate to the Planning Methods list in the model (typically in a SYS or ADMIN module, or via the list manager).
  2. 2Add a new list member: name = Headcount x Rate (match naming convention of existing methods).
  3. 3As best practice, assign a code to this method (e.g., HC_RATE). Codes make lookup formulas more stable than names.
  4. 4Save the list.
💡 Use Codes, Not Names

Formulas that reference the planning method should point to the code, not the display name. Display names can be changed by admins; codes stay stable.

✅ What it should look like when complete
C2
Associate Method with Business Area
⏱ 5 min
Objective Link the Headcount × Rate method to the Expense Planning business area only. This controls which modules and UX pages this method appears in.
  1. 1Open the SYS module for Planning Methods (e.g., SYS PLN Planning Methods or similar).
  2. 2Find the row for Headcount x Rate.
  3. 3Set Business Area → Expense Planning only.
  4. 4If the module has a description field, enter: Plans expenses using headcount × average rate per person + adjustment. This text displays to end users in the UX.
  5. 5Important: Do not set this to All Business Areas — it would cause the method to appear in Gross Profit, Balance Sheet, and Headcount modules where it is not appropriate.
  6. 6Save.
ℹ Business Area Scope

One planning method can apply to multiple business areas without creating duplicates. Direct Input, for example, is used in Gross Profit, Expense, and Balance Sheet planning — it's the same method entry, not three separate entries.

✅ What it should look like when complete
C3
Configure Module Line Items
⏱ 20 min
Objective Add the headcount, rate, and adjustment line items to the OpEx input module. Configure formulas, access drivers, and write rules correctly.
  1. 1Open the INP OPX Expense Planning module in blueprint mode.
  2. 2Add line item: HC Boolean — Boolean format. Formula: looks up whether the Headcount x Rate method is assigned to the current GL account in the planning methods mapping module.
  3. 3Add line item: HC Amount — Number format. Formula: IF HC Boolean THEN [lookup headcount from DAT module for this department] ELSE 0. Apply Access Driver based on HC Boolean.
  4. 4Add line item: Rate — Number format. Formula scope: actuals version only. Formula: [actuals GL balance for account] / HC Amount (gives actual rate). For forecast: user-input. Apply Access Driver based on HC Boolean. Set Write Rule on forecast periods.
  5. 5Add line item: Adjustment — Number format. No formula (always input). Apply Access Driver based on HC Boolean. Set Write Rule.
  6. 6Modify Final Amount formula: add an additional IF branch for when HC Boolean is true: ELSEIF HC Boolean THEN HC Amount * Rate + Adjustment.
  7. 7Tag Final Amount with Formula Override and Extension data tags.
  8. 8Tag HC Amount, Rate, and Adjustment with Extension data tag.
  9. 9Save all changes.
⚠ Applies-To Configuration

The HC Boolean, HC Amount, Rate, and Adjustment line items should NOT have time-variant Applies-To for the Boolean — the planning method assignment doesn't change by period. Only Rate needs a formula scope restriction to actuals version. Getting Applies-To wrong causes excessive calculations across periods where the method is not active.

✅ What it should look like when complete
C4
Update the UX Filter Subset
⏱ 8 min
Objective Add the four Headcount × Rate line items to the OPEX filter subset so they appear in the UX when the method is active.
  1. 1Navigate to the OPEX filter line item subset.
  2. 2Open in edit mode.
  3. 3Add: HC Amount, Rate, Adjustment.
  4. 4The subset controls what line items surface in the UX planning grid. Without this step, the new line items will not appear to users.
  5. 5Save.
✅ What it should look like when complete
C5
Update Admin Line Item Subset
⏱ 8 min
Objective Map the new line items to the Headcount × Rate method in the admin subset so they display correctly in the top-level admin configuration page.
  1. 1Navigate to the Admin line item subset (ADMIN OPX module).
  2. 2Find Final Amount row. Note its method association for the Headcount x Rate method.
  3. 3Copy that method association to: HC Amount, Rate, Adjustment.
  4. 4The combination of filter subset + admin subset controls what the end user sees — both are required.
  5. 5Save.
✅ What it should look like when complete
C6
Map Method to the Training GL Account
⏱ 5 min
Objective Assign the Headcount × Rate method to the Training GL account in the planning methods mapping module.
  1. 1Navigate to the Planning Methods mapping module (typically ADMIN OPX Planning Methods Assignment or similar).
  2. 2Find the Training GL account row.
  3. 3Change the Planning Method from the current method (e.g., Units × Rate) to Headcount x Rate.
  4. 4Save.
✅ What it should look like when complete
C7
Test End to End
⏱ 10 min
Objective Verify the Headcount × Rate method works correctly for the Training account in the UX — actuals show actual rate, forecast allows input, total calculates correctly.
  1. 1Navigate to the UX → Operating Expenses planning page.
  2. 2Select or filter to the Training GL account.
  3. 3In actuals periods: verify HC Amount shows actuals headcount, Rate shows actuals GL ÷ headcount, Adjustment = 0, Final Amount = correct.
  4. 4In forecast periods: verify HC Amount shows headcount from the Headcount model (read-only), Rate allows input.
  5. 5Enter a Rate value in a forecast period. Enter an Adjustment value.
  6. 6Verify Final Amount = HC Amount × Rate + Adjustment. Check the math manually.
  7. 7Switch to a different GL account (one using a different planning method). Verify the HC Amount, Rate, Adjustment line items are hidden (access driver working correctly).
✅ What it should look like when complete

Debrief Questions

  2. What happens if headcount data is not loaded for a forecast period — what does the user see?
  3. How would you add a third adjustment layer to this method?
  4. What is the upgrade risk of this extension? Which specific line item carries the highest risk?
