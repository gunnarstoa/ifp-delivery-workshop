🔍 生成後の検証 — ワークショップと実務の違い

このページは、モデルを機能させるために必須となる優先度の高いチェックを扱います。

実際の本番導入では、生成後の検証はもっと広範囲です——通常は1〜2日かけて体系的にレビューし、以下を網羅します：

  • 構成の検証（階層構造、ディメンション設定）
  • セキュリティとアクセスの検証
  • パフォーマンス最適化と数式の効率
  • ドキュメント整備と引き継ぎ準備
  • データロードの検証（実績データ連携）
  • ユーザー受け入れテストシナリオ

完全な生成後チェックリストはConfiguration GuideのSection 5および付録Aに記載されています。

本ワークショップは中核となる概念とワークフローに焦点を当てています。実プロジェクト導入では、必ずConfiguration Guideの完全なチェックリストに従ってください。

生成のたびに必ず実施すること

生成が完了したら、他の作業を始める前に以下のチェックリストを実行してください。これにより、データや拡張機能を追加する前に、完全で安定したベースラインが確保されます。

  • 4つのモデルがすべてワークスペースに存在することを確認する（Financial Planning、Headcount、Admin、CapEx）
  • アプリケーションオブジェクトの個数を確認する — Application Frameworkの概要に、モデル、UXアプリ、アセット、Data Orchestratorオブジェクトが表示されます
  • Error Logを開き、エラー総数を記録する — これをベースラインとして残します
  • Error Logを確認し、すべてのモジュールが正しく生成されたことを検証する — モジュールが欠落している場合は、ここにエラーとして現れます。詳細はError Logレビューセクションを参照してください。
  • 主要なSYSモジュールのラインアイテムが投入されていることを確認する：Planning Methodsリストは事前投入された状態で生成されます。すべての本番データリスト（エンティティ、部門、口座）は未投入の状態で生成され、Data Orchestratorからデータをプッシュすることで投入されます。
  • 階層リストに想定どおりのメンバーがロードされていることを確認する。すべての階層リストメンバーはADO経由で投入されます——一部はCSVまたは接続された外部ソース経由です。エンティティ、部門、IS Accounts、BS Accountsなどの階層では、ソースリストメンバーをプランニング側の同等項目にマッピングするため、他のスポークモデルへプッシュする前にAdminモデルで追加のマッピング手順が必要です。
  • Planning Methodリストのメンバーが投入されていることを確認する — この段階で検証可能なのはこの項目のみです。Planning Methodsに紐づくSYSおよびADMINモジュールは後から投入されます。この手順を完了するには、Configuration Guideの付録A3を参照してください。
  • CALCモジュール内の計算ラインアイテムをスポットチェックする — 数式が空ではないことを確認します
  • ▶ 動画で見る：IFPv2 モデル生成後の作業 (IFPv2 Models Post Generation) — 上記の生成後検証ステップを扱う短い動画です。（SharePointで開きます · Team-JAPAN パートナーサイトへのアクセス権が必要です）
⚠ 数式が空ですか？まずError Logを確認してください

本来あるはずの数式が空になっている場合、最も確実な検証方法はError Logを確認することです。すべての数式エラーは数式を空（BLANK）にします——参照無効が原因で失敗しており、ラインアイテム名の変更が認識されていないか、構造変更が原因です。数式が欠けていると決めつける前に、必ずError Logを確認してください。

生成後のミニ演習

1
生成したモデルを検証する
⏱ 20分
目的 生成後チェックリストを一通り実施します。ベースラインのエラー件数を把握し、モデル構造が完全であることを確認します。
  1. 1ご自身のワークスペースで、4つのモデルが一覧に表示されていることを確認します。メモに名前を記録します。
  2. 2Application Frameworkの概要ページを開きます。記録項目：モデル数、UXアプリ数、アセット数、エラー数。
  3. 3右下のError Logsをクリックします。エラー総数を記録します — これがベースラインです。
  4. 4Financial Planningモデルを開きます。Modulesに移動します。各プレフィックス（ADMIN、SYS、DAT、INP、CALC、OUT）ごとに少なくとも1つのモジュールが存在することを確認します。
  5. 5SYSモジュールを開きます。Planning Methodsリストを探します。Application Framework Configuratorで有効化したメソッドが存在することを確認します。
  6. 6CALCモジュール（例：CALC OPX IS Itemizedなど）を開きます。Blueprintでラインアイテムを開きます。数式フィールドが空でないことを確認します。
  7. 7チェックリスト完了をマークします。Error Logレビューに進む準備が整いました。
ℹ 完全な生成後チェックリスト

完全な生成後検証チェックリストは、Configuration GuideのSection 5および付録Aを参照してください。
