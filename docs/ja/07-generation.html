生成で何が行われるか

生成を実行すると、Application Frameworkは44の構成質問への回答をすべて読み取り、完全なIFPモデル構造をプロビジョニングします。これはテンプレートのコピーではなく、構成上の意思決定を反映して回答内容に応じたモデルを生み出す動的な生成プロセスです。

生成によって、以下の4つのモデルがプロビジョニングされます：

  • Financial Planning — メインのプランニングモデル（INP、CALC、OUTモジュール）
  • Headcount — HCモデル、Jobレベルのプランニング（有効化されている場合）
  • Admin — 口座階層、エンティティ、部門のソースからプランニングへのメタデータマッピングを行う主要な場所です。
  • CapEx — capexプランニングモデル（有効化されている場合のみ）

生成のステータス

ステータス意味対応
Generating...生成が進行中です。モデルがプロビジョニングされています。そのまま待ちます。別の生成を実行しないでください。想定所要時間：10〜20分。
Generated with Errors生成は完了しました。ベースモデルとの構成差異により、一部の数式にエラーがあります。これは正常かつ想定内です。Error Logを確認してください——ほとんどのエラーは解決可能です。
Generated Successfully数式エラーが検出されることなく生成が完了しました。生成後のステップに進んでください。
Generation Failedプラットフォームエラーにより生成が完了しませんでした。Anaplanサポートに連絡してください。原因を特定せずに再生成を試みないでください。
🚨 「Generated with Errors」は壊れているという意味ではありません

「Generated with Errors」は正常かつ想定内であり、特にトレーニングや初期導入ではよく見られます。これは、ベースモデルテンプレートとは異なるアイテムを参照する数式があることを意味します——通常は階層名の変更、地域の削除、またはテンプレート前提とは異なる構成上の選択が原因です。モデルは機能します。Error Logレビューのセクションでエラーリストを体系的に処理していきます。

生成のステップ

1

生成を実行する

Application Framework Configuratorからすべてのセクションを完了したら、Generateをクリックします。プロンプトが表示されたら確認します。開始時刻を記録します。

2

ステータスを監視する

Application Frameworkの概要ページに生成ステータスが表示されます。定期的に更新してください。進行中にブラウザを閉じたり、別の生成を実行したりしないでください。

3

生成されたモデルを最初に確認する

完了したら、ご自身のワークスペースに移動します。4つのIFPモデルが一覧に表示されているはずです。Financial Planningモデルを開き、構成内容に応じてプランニングのサブプロセス（Revenue & COGS、Operating Expense、Headcount、CapEx Planning、Balance Sheet & Cash Flow）が存在することを確認します。

4

Error Logを確認する

Application Frameworkに戻ります。概要ページの右下にError Log件数があります。クリックしてログを展開します。注：エラーが0件以上あるのは正常です。Error Logレビューに進んでください。
