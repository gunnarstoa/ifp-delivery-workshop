今後のIFP機能

1

CapExプランニングアプリケーション

「今年後半に、CapExプロジェクト向けのアプリケーションをリリース予定です。今後6か月以内にお客様案件がある場合は、当面は個別対応となります。」専用設計のCapExプランニングアプリケーションがIFPロードマップに含まれています。利用可能になるまでは、CapEx要件はカスタム拡張機能、または別のAnaplan構築として対応します。

2

ADOの進化

Anaplan Data OrchestratorはIFP 2.0において投資が継続される見込みです。具体的には、一般的なERPソース向けの事前構築データパイプライン、およびパイプライン実行時のエラー報告改善が挙げられます。現在のIFPにおけるADOセットアップは、多くのお客様環境で手動でのパイプライン構成が必要です。

3

Headcount Planning 2.0ロードマップ

IFP 2.0でリリースされたHeadcount Planning 2.0は反復改善が進められています。計画されている拡張には、追加のプランニングメソッドタイプや、ワークフォースプランニングシナリオとのより緊密な統合が含まれます。リリースノートはアンプラン・コミュニティでご確認ください。

4

Application Frameworkの拡充

Application Framework Configuratorは構成カバレッジの拡大が見込まれており、構成で対応可能な要件にもかかわらずカスタム拡張機能が必要となるケースを削減していきます。IFPのリリースごとに新しい構成質問が追加されてきました。
