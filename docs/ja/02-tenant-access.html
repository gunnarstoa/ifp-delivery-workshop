開始前に

本ワークショップでは、IFP 2.1モデルを事前準備した参加者ごとの専用トレーニング用テナントを使用します。先に進む前に、正しいテナントとワークスペースへのアクセスを必ず確認してください。本番テナントは使用しないでください。

⚠ 重要

以下の手順でテナントアクセスを確認できない場合は、続行する前にワークショップのSlackチャンネルでお問い合わせください。すべての実践演習はこのアクセスが整っていることが前提となります。

ステップ1 — ログイン

https://us1a.app.anaplan.com/ にアクセスし、テナントアクセス案内のメールに記載されたログイン情報でサインインしてください。

ステップ2 — 正しいテナントを確認

ログイン後、正しいトレーニング用テナントにいることを確認してください。テナント名は上部ナビゲーションバーに表示されます。

ステップ3 — ロールとDataspaceの確認

Administrationセクションに移動し、生成に必要な4つのロールがすべてユーザーに付与されていること、およびDataspaceが割り当てられていることを確認してください。これを生成の前に必ず行う習慣をつけてください——パートナーがプロジェクトの途中で詰まる最も一般的な原因です。

ユーザーには、以下の4つすべてのロールが必要です：

  • Application Owner
  • Workspace Admin
  • Page Builder
  • Integration Admin

加えて、ユーザーにDataspaceが割り当てられていることも確認してください。これがないと生成は失敗します。

⚠ なぜこれが重要か

これらのアクセス権をテナント管理者と事前に確認せずに生成を試みた結果、実は権限の問題だった生成失敗のデバッグに何時間も費やすパートナーをよく見かけます。構成を始める前に、必ずロールとDataspaceの割り当てをテナント管理者と確認してください。生成前の習慣にしましょう。

事前準備済みのTRAINモデルについて

お使いのトレーニング用テナントには、ワークショップ演習で各自が構成するものとは別に、Anaplanが提供する一連のモデルとアプリケーションも含まれています。これらはTRAINプレフィックス（例：IFPv2.0.0-train-base-financial-planning）で命名されており、リアルなサンプルデータが事前に投入されています。

⚠ 拡張機能演習にはTRAINモデルを使用してください

ワークショップの拡張機能セクションに到達した際は、各自の構成から生成したモデルではなく、事前準備済みのTRAINモデルに対して拡張機能演習を実施してください。TRAINモデルにはデータが投入済みであり、拡張機能の動作をエンドツーエンドで実証・検証するために必要です。各自が構成したモデルは、構成と生成の演習専用とお考えください。

準備完了です

正しいトレーニング用テナントにいること、必要な4つのロールを持っていること、Dataspaceが割り当てられていること、どの演習でどのモデルを使うかが分かったら、セットアップは完了です。IFP概要に進み、ワークショップの本編を始めてください。
