Configuration Exercises 1-3

Hands-on exercises to configure three different IFP scenarios

Day 190 minGuided Practice
📖 Tell Exercise Overview Each exercise presents a real business scenario with configuration requirements.
👁 Show Facilitated Walk-Through Facilitator guides participants through configuration decisions step-by-step.
✅ Do Configure It Yourself Participants configure their own version, comparing results to the example.
💡 Key Rules for Configuration
  • Renames should be NO LONGER than the original dimension name, particularly the 'Department' dimension
  • All standard dimensions are composite hierarchies — custom/flat lists are handled via extension
  • Minimum levels per dimension: 2 | Maximum: 8 (soon to be increased to 12)
  • When modifying hierarchy levels: add/remove from the middle, never remove the leaf level
  • Always click "Apply Changes" after each modification
  • Use "Restore default" to reset if needed

Exercise 3: Nuclear Plant – OpEx and Headcount

Scenario: A utilities company operating a nuclear facility needs operating expense and headcount planning with a detailed work breakdown structure (WBS) for capital project tracking. No balance sheet, cashflow, or margin planning required.

Model Setup

  • Operating Expense and Headcount only
  • Will NOT use: top-down targets, margin planning, CapEx, balance sheet, cashflow, or allocations
  • Will configure extension for CapEx projects & PO tracking
  • Time strata: Monthly
  • Generation time: TBD

Key Configuration Decisions

  • Functional Area repurposed as WBS (Work Breakdown Structure) with 5 levels for detailed project hierarchy
  • Multiple custom dimensions for project and procurement tracking (handled in extension)
  • Rich attribute structure on Cost Center for multi-dimensional analysis
  • Minimal scope: OpEx + Headcount only, keeping configuration simple
⚠ Custom Dimensions Are Extensions

PO Headers, PO Items, and other custom lists for this use case are NOT part of the base generation. They will be added as part of the extension phase, after the model is generated with the core dimensions configured.
