The verification checklist to run after every generation before any other work

Required After Every Generation

After generation completes, run through the following checklist before doing any other work. This ensures you have a complete, stable baseline before adding data or extensions.

  • Verify all 4 models exist in the workspace (Financial Planning, Headcount, Admin, Data Orchestrator)
  • Check application objects count — App Framework overview shows: models, UX apps, assets
  • Open error logs and note total error count — record this as your baseline
  • Open the Financial Planning model and verify module list shows ADMIN, SYS, DAT, INP, CALC, OUT prefixes
  • Verify key SYS module line items are populated: planning methods list, functional area list, entity hierarchy
  • Confirm hierarchy lists loaded with expected members (check the entity list for FintechCo's 3 entities)
  • Check that enabled planning methods are visible in ADMIN configuration module
  • Run a spot-check on a calculated line item in a CALC module — confirm formula is not blank
💡 Export Error Logs to Excel

If you have more than 10 errors, export the error log to Excel before reviewing in-app. Filtering and categorizing errors is much faster in Excel than in the App Framework log view. Sort by Error Type first, then by Model to group related errors.

Post-Generation Mini-Lab

1
Verify Your Generated Model
⏱ 20 min
Objective Work through the post-generation checklist on your FintechCo model. Identify the baseline error count and confirm the model structure is complete.
  1. 1From your workspace, confirm 4 models are listed. Name them in your notes.
  2. 2Open the App Framework overview page. Record: model count, UX app count, asset count, error count.
  3. 3Click on Error Logs (bottom-right). Record the total error count — this is your baseline.
  4. 4Open the Financial Planning model. Navigate to Modules. Verify at least one module exists for each prefix: ADMIN, SYS, DAT, INP, CALC, OUT.
  5. 5Open a SYS module. Find the Planning Methods list. Verify the methods you enabled in the configurator are present.
  6. 6Open the Entity list in the model. Verify FintechCo's 3 entities are present (or will be loaded via ADO).
  7. 7Open a CALC module (e.g., CALC OPX IS Itemized or similar). Open a line item in blueprint. Verify the formula field is not blank.
  8. 8Mark your checklist complete. You are ready to review error logs.
