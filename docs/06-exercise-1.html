Configuration Exercise 1: OpEx and Headcount Planning

Hands-on exercise to configure IFP for a manufacturing company

Configuration30 minGuided Practice
💡 Key Rules for Configuration
  • Use a standardized naming convention for all generated models and apps — every model and app name must be unique within the tenant. Duplicate names will cause generation to fail. See the example pattern below.
  • Renames should be NO LONGER than the original dimension name, particularly the 'Department' dimension
  • All standard dimensions are composite hierarchies — custom/flat lists are handled via extension
  • Minimum levels per dimension: 2 | Maximum: 15
  • When modifying hierarchy levels: add/remove from the middle, never remove the leaf level
  • Always click "Apply Changes" after each modification
  • Use "Restore default" to reset if needed

Standardized Naming Convention

Replace the default model and app names with a standardized format before generating. The recommended pattern is [ModelType]_Exercise ##_YYYYMMDD — this ensures every generation produces uniquely-named artifacts and avoids name-collision generation errors when the same exercise is run more than once in the tenant.

Scenario

A manufacturing company needs to plan operating expenses and headcount across multiple departments and cost centers. No revenue, COGS, balance sheet, or capital expenditure are required.

Model Setup

  • Operating Expenses (Financial Planning Model)
  • Headcount Planning
  • No Revenue/COGS, Balance Sheet, or CapEx
  • Estimated generation time: 20 minutes
