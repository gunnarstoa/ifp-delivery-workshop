Qué es IFP

Integrated Financial Planning (IFP) es la aplicación pre-construida de Anaplan que habilita casos de uso fundacionales de FP&A para planeación, presupuestación y forecasting de ingresos, gastos operativos, gastos de capital, estados financieros integrados y más, apoyando la toma de decisiones con confianza y el monitoreo continuo de la salud financiera. Proporciona un enfoque estructurado de planeación financiera, basado en el Application Framework Configurator, que elimina meses de tiempo de construcción de modelos y entrega a los consultores partner un marco de delivery repetible.

La versión 2.1 es el release actual. Se basa en la re-arquitectura de la 2.0 que introdujo el Application Framework: un sistema de generación de modelos basado en el Application Framework Configurator que reemplaza la configuración manual de modelos, junto con la integración del Anaplan Data Orchestrator (ADO) y Headcount Planning 2.0. La versión 2.1 refina la configuración de ADO e incrementa los límites de dimensionalidad.

ℹ Rangos de niveles

El valor por defecto son 3 niveles por Dimensión, excepto Job que por defecto es 4. Cada Jerarquía puede reducirse a un mínimo de 2 niveles, o aumentarse hasta 15 niveles por Dimensión. Al eliminar niveles, el nivel hoja original siempre debe permanecer en la parte inferior. Al añadir niveles, el nuevo nivel se añade en la parte inferior y debe arrastrarse por encima del hoja original.

Requisitos de aprovisionamiento

Antes de que cualquier implementación pueda comenzar, los siguientes pasos de aprovisionamiento deben completarlos el equipo Anaplan:

  1. El cliente compra la aplicación IFP (incluye una tarifa de suscripción que cubre actualizaciones)
  2. El Customer Success Business Partner abre un ticket de aprovisionamiento solicitando la creación de un Workspace (mínimo 100GB)
  3. Para entornos ALM (dev/QA/prod), se necesitan tickets de aprovisionamiento de Workspace separados para cada entorno
  4. El equipo Anaplan despliega el Application Framework en el Tenant del cliente
  5. El partner y el Customer Success Business Partner son notificados cuando el despliegue está completo
⚠ Requisito de Tenant Admin para la generación

El Application Framework solo puede generar desde su Tenant por defecto. Un partner usando el correo de su firma (por ej., john@partnerfirm.com) no puede generar si el Tenant del cliente no es el suyo por defecto. Solución: obtener una dirección de correo en el dominio del cliente, o que el Tenant Admin del cliente genere mientras usted lo guía. Roles requeridos para la generación: Application Owner, Workspace Admin, Page Builder, Integration Admin.

El concepto del Application Framework

El Application Framework es el motor de configuración y generación. El flujo de trabajo es:

1

Configurar

Responder todas las preguntas de configuración en el Application Framework Configurator. Estas definen los niveles de Jerarquía, las Dimensiones, los métodos de planeación y el alcance.

2

Generar

Disparar la generación. El Application Framework lee sus respuestas y aprovisiona los cuatro modelos IFP (Financial Planning, Headcount, Admin, CapEx). Esto toma 20–40 minutos.

3

Revisar

Inspeccionar el modelo generado y los Error Logs. 'Generated with errors' es normal y esperado: no significa que el modelo esté roto.

4

Extender

Añadir extensiones específicas del cliente sobre el modelo base generado. Documente todo lo que cambie.

🚨 'Generated with Errors' es normal

Cada generación de IFP 2.1 en un entorno de entrenamiento o etapa temprana produce errores. Estos son errores de fórmula causados por las elecciones de configuración, el renombrado de Jerarquías o diferencias geográficas respecto a la plantilla del modelo base. El modelo es funcional. Aprenderá a leer y resolver estos errores en la sección Revisión del Error Log.
