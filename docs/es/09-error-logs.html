Revisión del Error Log

Cómo leer, categorizar y resolver los Error Logs de generación

Día 130 minPráctico

Error Logs - qué son

Los Error Logs son generados por el App Framework después de la generación del modelo. Listan problemas de fórmula y configuración que surgieron cuando el modelo generado difiere de la plantilla del modelo base. El Error Log es accesible desde la parte inferior derecha de la página de resumen del App Framework y puede exportarse a Excel.

Los Error Logs son contenido nuevo introducido en IFP 2.0: las versiones anteriores de IFP no tenían este reporte sistemático de errores. Aprender a leerlos eficientemente es una habilidad clave del practicante.

Referencia de tipos de error

Tipo de errorQué significaAcción
LINE_ITEM_FORMULAUna fórmula referencia un Line Item que no existe o fue renombrado en el modelo generado, a menudo debido al renombrado de Jerarquías o elecciones de configuraciónEncuentre el Line Item por código (prefijo de guión bajo), compare la fórmula con el modelo base, corrija o copie-ajuste desde el base
LIST_PROPERTY_FORMULALa fórmula referencia una propiedad de Lista que no existe o fue renombradaEncuentre el Line Item por código, compare la fórmula con el modelo base, actualice la referencia a la propiedad de Lista
IMPORT_ACTIONUna import action referencia un Módulo source o una Lista que cambióActualice el mapeo de importación en el modelo afectado
MODEL_DATA_SOURCEUna conexión de fuente de datos está rota (referencia de pipeline ADO)Re-establezca la conexión en ADO o actualice la referencia a la fuente de datos

Cómo encontrar un error de fórmula

El tipo de error más común es LINE_ITEM_FORMULA. Aquí está el flujo de trabajo de diagnóstico:

"Generalmente le decimos el Line Item con un guión bajo al inicio: ese es el código de Line Item del error. Vaya al modelo, busque ese código y lo lleva al elemento exacto."
— Facilitador del taller, Día 1
1

Abrir el Error Log

Desde el resumen del App Framework, haga clic en el panel del Error Log (parte inferior derecha). Filtre solo a errores de fórmula. Si hay más de 10 errores, exporte a Excel primero.

2

Anote el código

Cada error incluye un código de Line Item en el formato _LINEITEMNAMEEXCEPTION (prefijo de guión bajo). Este código es único: úselo para la búsqueda en el modelo, no el nombre de visualización.

3

Encuentre el Line Item afectado

Vaya al modelo afectado (el Error Log le dice qué modelo). Abra Line Items. Busque el código con el prefijo de guión bajo. El resultado de búsqueda lo lleva directamente al Módulo y al Line Item.

4

Compare con el modelo base

Abra el modelo base (su facilitador le proporcionará acceso). Encuentre el mismo Line Item. Compare la fórmula generada con la del modelo base. Identifique qué cambió: renombrado de Jerarquía, eliminación de geografía, cambio de Dimensión.

5

Corregir o copiar-ajustar

Opción A: corrija la fórmula directamente, actualice las referencias para coincidir con su configuración. Opción B: copie la fórmula del modelo base a su modelo y ajuste las referencias de Dimensión. La Opción B es más segura y rápida para la mayoría de errores LINE_ITEM_FORMULA.

6

Validar

Después de corregir, revise si la fórmula se resuelve (sin indicador rojo de error de fórmula). Vuelva al Error Log y marque el error como resuelto. Proceda al siguiente error.

7

Escalar si está atascado

Si el error parece ser un problema a nivel de plataforma (no causado por su configuración), registre el código de error y contacte al soporte Anaplan. No pierda tiempo en errores de plataforma durante un compromiso de delivery.

La regla del 90%

ℹ Espere errores, entregue sistemáticamente

Cada configuración de cliente producirá un conjunto diferente de errores. La meta no es cero errores antes de proceder: es llegar del 0% al 90% rápidamente. Categorice los errores, corrija los resolubles sistemáticamente y registre los restantes como una lista de problemas conocidos. "Ese último 10% es lo que se trabaja metódicamente", no una señal de que el proyecto está roto.

Ejercicio de Error Log

1
Ejercicio del Error Log
⏱ 30 min
Objetivo Encuentre todos los errores en el modelo generado. Categorice cada uno por tipo de error. Identifique la causa raíz de al menos un error LINE_ITEM_FORMULA y proponga una corrección.
  1. 1Desde el App Framework, abra el Error Log. Exporte a Excel si tiene más de 10 errores.
  2. 2Cree una tabla simple: código de error | tipo de error | modelo | causa raíz | acción
  3. 3Para cada error, categorícelo como: LINE_ITEM_FORMULA, LIST_PROPERTY_FORMULA, IMPORT_ACTION o MODEL_DATA_SOURCE.
  4. 4Elija el primer error LINE_ITEM_FORMULA. Anote el código (prefijo de guión bajo).
  5. 5Vaya al modelo afectado. Busque el código en Line Items. Confirme que encuentra el Line Item correcto.
  6. 6Abra el Line Item en blueprint. Registre la fórmula actual (rota).
  7. 7Abra el modelo base. Encuentre el mismo Line Item. Registre la fórmula del modelo base.
  8. 8Compare las dos fórmulas. Identifique qué referencia cambió (¿renombrado de Jerarquía? ¿geografía? ¿Dimensión?).
  9. 9Corrija la fórmula en su modelo O copie-ajuste desde el modelo base. Confirme que se resuelve.
  10. 10Vuelva al Error Log. ¿Cuántos errores quedan? ¿Qué errores son seguros de dejar por ahora?

Preguntas de cierre

  1. ¿Qué errores son seguros de dejar por ahora? ¿Cuáles deben corregirse antes de las pruebas?
  2. Si un cliente pregunta "¿está el modelo listo para usar?" y tiene 8 errores en el log, ¿qué dice?
  3. ¿Cuándo escala un error al soporte Anaplan vs. lo corrige usted mismo?
  4. ¿Cómo documentaría su enfoque de resolución de errores para el siguiente consultor en este proyecto?
