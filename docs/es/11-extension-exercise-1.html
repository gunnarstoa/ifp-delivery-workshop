Ejercicio de extensión 1: ajustes manuales

Añada 2 nuevos Line Items de ajuste manual al Módulo OpEx y modifique la fórmula de agregación

Extensiones30 minPráctica guiada
🌐 Aviso de traducción

Esta es una traducción automática de la versión original en inglés. Para la versión autoritativa y más actualizada, consulte la versión en inglés. Reporte cualquier problema de traducción a su contacto Anaplan.

Requisito de negocio

Se necesitan 2 tipos adicionales de ajustes manuales para inputs de gastos.

Solución

  • Añadir nuevos Line Items al Módulo afectado
  • Modificar la fórmula de agregación para incluir los nuevos Line Items

Pasos

1

Abrir el Módulo Expense Planning

Abra el Módulo INP OPX Expense Planning en el modelo "Financial Planning".

2

Añadir 2 nuevos Line Items

Añada 2 nuevos Line Items en la sección "Direct Input Method" del blueprint:

  • Manual Adjustment #1
  • Manual Adjustment #2
  • Establezca el Format de los nuevos Line Items (igual que "Amount")
  • Establezca el Write Access Driver para cada nuevo Line Item
  • Documente para qué son los Line Items en el campo "Notes"
3

Modificar la fórmula de agregación

Necesitamos modificar la fórmula para el Line Item Direct Input Fcst para incluir los 2 nuevos Line Items:

  • Modifique la fórmula
  • Documente lo que ha hecho:
    • Use el campo "Notes" del blueprint del Módulo
    • Mantenga una lista maestra de overrides en un documento offline, incluyendo la sintaxis de fórmula original y actualizada
4

Actualizar el Line-Item Subset y los ajustes Admin

Añada los 2 nuevos Line Items al LISS (Line-Item Subset for Show/Hide):

  • Abra el line-item subset "LISS: OpEx Filter"
  • Marque el booleano para los 2 nuevos ajustes
  • Abra el Módulo "ADMIN LISS OpEx Filter" y establezca el "Forecast Method" a "Direct Input" para cada uno de los 2 nuevos Line Items
5

Verificar en la UX App

Abra la página "OpEx – planning" en la UX App:

  • Seleccione la cuenta "6005 Office Equipment"
  • Revise que la lógica funcione correctamente
