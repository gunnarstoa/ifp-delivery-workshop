🔍 Verificación posterior a la generación: taller vs. realidad

Esta página cubre las verificaciones esenciales de alta prioridad para un modelo funcional.

En una implementación real de producción, la validación posterior a la generación es mucho más extensa: típicamente 1–2 días de revisión sistemática que cubren:

  • Verificación de configuración (estructura de Jerarquía, ajustes de Dimensiones)
  • Validación de seguridad y acceso
  • Optimización de rendimiento y eficiencia de fórmulas
  • Documentación y preparación de entrega
  • Validación de carga de datos (integración de actuals)
  • Escenarios de pruebas de aceptación de usuario

La lista de verificación completa posterior a la generación está en la Sección 5 y el Apéndice A de la Configuration Guide.

Este taller se enfoca en los conceptos centrales y el flujo de trabajo. Para implementaciones en vivo, siempre siga la lista de verificación completa de la Configuration Guide.

Requerido después de cada generación

Después de que la generación complete, recorra la siguiente lista de verificación antes de hacer cualquier otro trabajo. Esto asegura que tiene una línea base completa y estable antes de añadir datos o extensiones.

  • Verifique que los 4 modelos existan en el Workspace (Financial Planning, Headcount, Admin, CapEx)
  • Revise el conteo de objetos de aplicación: el resumen del App Framework muestra: modelos, UX apps, activos, objetos de data orchestrator
  • Abra los Error Logs y anote el conteo total de errores: registre esto como su línea base
  • Revise el Error Log para verificar que todos los Módulos se generaron correctamente: cualquier Módulo faltante surgirá aquí como error. Vea la sección Revisión del Error Log para detalles.
  • Verifique que los Line Items clave de los Módulos SYS estén poblados: la Lista Planning Methods vendrá pre-poblada. Todas las Listas de datos de producción (entity, department, accounts) vendrán sin poblar y se poblarán al publicar datos desde Data Orchestrator.
  • Confirme que las Listas de Jerarquía cargaron con los miembros esperados. Todos los miembros de Listas de Jerarquía se poblan vía ADO, algunos vía CSV o fuente externa conectada. Jerarquías como entity, department, IS Accounts y BS Accounts requieren un paso de mapeo adicional en el Admin Model para mapear los miembros de la Lista source a sus equivalentes de planeación antes de publicar a otros modelos spoke.
  • Revise que los miembros de la Lista Planning Method estén poblados: este es el único elemento verificable en esta etapa. Los Módulos SYS y ADMIN ligados a Planning Methods se pueblan después; consulte el Apéndice A3 de la Configuration Guide para completar este paso.
  • Ejecute una verificación puntual en un Line Item calculado en un Módulo CALC: confirme que la fórmula no esté en blanco
⚠ ¿Fórmulas en blanco? Revise primero el Error Log

La forma #1 de verificar si una fórmula debería estar y resulta en blanco es revisar el Error Log. Todos los errores de fórmula harán que la fórmula esté EN BLANCO: fallan debido a una referencia inválida, ya sea por cambios de nombre de Line Item no reconocidos o cambios de estructura. Siempre revise el Error Log antes de suponer que una fórmula falta.

Mini-ejercicio posterior a la generación

1
Verifique su modelo generado
⏱ 20 min
Objetivo Recorra la lista de verificación posterior a la generación. Identifique el conteo de errores de línea base y confirme que la estructura del modelo está completa.
  1. 1Desde su Workspace, confirme que 4 modelos están listados. Nombrelos en sus notas.
  2. 2Abra la página de resumen del App Framework. Registre: conteo de modelos, conteo de UX apps, conteo de activos, conteo de errores.
  3. 3Haga clic en Error Logs (parte inferior derecha). Registre el conteo total de errores: esta es su línea base.
  4. 4Abra el modelo Financial Planning. Navegue a Modules. Verifique que exista al menos un Módulo para cada prefijo: ADMIN, SYS, DAT, INP, CALC, OUT.
  5. 5Abra un Módulo SYS. Encuentre la Lista Planning Methods. Verifique que los métodos que habilitó en el Application Framework Configurator estén presentes.
  6. 6Abra un Módulo CALC (por ej., CALC OPX IS Itemized o similar). Abra un Line Item en blueprint. Verifique que el campo fórmula no esté en blanco.
  7. 7Marque su lista de verificación como completa. Está listo para revisar los Error Logs.
