Lab B: Direct Input + Adjustments

Build a two-layer adjustment extension on the Direct Input planning method

Lab B45 minHands-On
📖 Tell The Business Requirement The customer requirement: direct input + two adjustment layers for OpEx accounts.
👁 Show Facilitator Builds Live Facilitator builds the extension live before participants replicate it.
✅ Do Build the Extension Participants replicate the Direct Input + Adjustments extension in their FintechCo model.

Business Requirement

FintechCo's budget holders want to plan certain OpEx accounts using direct input — but with two adjustment layers on top: a management override and a regional adjustment. The formula is:

Direct Input + Adjustment 1 + Adjustment 2 = Total

The IFP base model supports Direct Input as a single-value method. Adding two adjustment layers requires extending the model. This is Lab B.

ℹ Model to Use

Work in the model your facilitator specifies — the IFPv2.0.0 train-base-financial-planning model (or similar). Do not work in your newly generated FintechCo model for this lab.

What You'll Build

Five changes to the IFP base model, in this exact order:

  1. Create data tags (Extension + Formula Override)
  2. Add Adjustment 1 and Adjustment 2 line items
  3. Modify the Final Amount formula to include the adjustments
  4. Update the OPEX filter line item subset
  5. Update the Admin line item subset
B1
Create Data Tags
⏱ 5 min
Objective Create the Extension and Formula Override data tags before making any model changes. These tags are your in-model record of every change you make.
  1. 1From the Financial Planning model, navigate to Settings → Data Tags (or Model Settings → Manage Tags).
  2. 2Create a new tag: name = Extension. Save.
  3. 3Create a second tag: name = Formula Override. Save.
  4. 4These tags are now available on all line items in this model.
💡 Why Tags First

If you build extensions first and add tags later, you'll have to hunt down every line item you touched. Creating tags first takes 5 minutes and saves you from hours of archaeology later — especially after an upgrade.

✅ What it should look like when complete
B2
Add Adjustment Line Items
⏱ 12 min
Objective Add Adjustment 1 and Adjustment 2 line items to the target OpEx input module. Configure each with the correct access driver so they only appear when the Direct Input method is active.
  1. 1Open the INP OPX Expense Planning module (or equivalent input module for OpEx) in blueprint mode.
  2. 2Find the Direct Input Forecast line item. This is the base line item you are extending.
  3. 3Add a new line item below it: name = Adjustment 1. Format = Number. Applies To = same as Direct Input Forecast.
  4. 4Copy the Access Driver (DCA) from Direct Input Forecast to Adjustment 1. This ensures Adjustment 1 only shows when the Direct Input method is active for the account.
  5. 5Set Write Rule on Adjustment 1: writable by users.
  6. 6Repeat steps 3–5 for Adjustment 2.
  7. 7Tag both new line items with the Extension data tag.
  8. 8Save and publish.
💡 Copy, Don't Recreate

Copy the line item settings from Direct Input Forecast rather than creating from scratch. Applies-To configuration, dimension settings, and time scope must match exactly.

✅ What it should look like when complete
B3
Modify the Final Amount Formula
⏱ 8 min
Objective Update the Final Amount formula to add Adjustment 1 + Adjustment 2 to the Direct Input value. Tag the modified line item as Formula Override.
  1. 1In blueprint mode, find the Final Amount line item in the same module.
  2. 2Open the formula. It currently calculates the total based on the active planning method.
  3. 3Find the section of the formula that handles the Direct Input method (look for an IF or CASE statement).
  4. 4Modify the Direct Input branch to add: + 'Adjustment 1'[ITEM: ITEM] + 'Adjustment 2'[ITEM: ITEM] (exact syntax may vary based on your model's line item names).
  5. 5Validate the formula — confirm it has no errors.
  6. 6Tag the Final Amount line item with Formula Override data tag.
  7. 7Also tag it with Extension data tag.
  8. 8Save.
⚠ Formula Override Risk

You have just modified a base model formula. If Anaplan releases an IFP update that changes this module, your formula override may be lost. The Formula Override tag is your marker for post-upgrade review. Document the formula change externally as well.

✅ What it should look like when complete
B4
Update the OPEX Filter Line Item Subset
⏱ 8 min
Objective Add the three relevant line items to the OPEX filter subset so they appear in the UX when the Direct Input method is active.
  1. 1Navigate to the OPEX filter line item subset (typically in a SYS or ADMIN module — ask facilitator if you can't find it).
  2. 2Open the subset in edit mode.
  3. 3Add the following line items to the subset: Direct Input Forecast, Adjustment 1, Adjustment 2.
  4. 4The subset controls which line items surface to users in the UX. All three must be included.
  5. 5Save the subset.
ℹ Filter Subset vs. Admin Subset

The filter subset controls UX visibility (what users see). The admin subset (next step) controls the top-level admin configuration page. Both need to be updated — they serve different purposes.

✅ What it should look like when complete
B5
Update the Admin Line Item Subset
⏱ 8 min
Objective Copy the planning method association from Final Amount to the three new line items so they display correctly in the admin configuration grid.
  1. 1Navigate to the Admin line item subset (in the ADMIN module — typically ADMIN OPX or similar).
  2. 2Find the Final Amount row in the admin grid.
  3. 3Note which planning method(s) Final Amount is associated with (specifically the Direct Input method entry).
  4. 4Copy that method association to Direct Input Forecast, Adjustment 1, and Adjustment 2.
  5. 5Save.
  6. 6Now test: navigate to the UX, open the Operating Expenses planning page, select a Direct Input account, refresh.
  7. 7Verify: Direct Input Forecast, Adjustment 1, and Adjustment 2 are all visible. Enter test numbers. Confirm Total = Direct Input + Adj1 + Adj2.
💡 Test Before Declaring Done

Enter actual numbers in all three fields and verify the math. A formula error could produce a result that looks correct but is actually ignoring one of the adjustments. Always put numbers in and verify the sum.

✅ What it should look like when complete

Debrief Questions

  1. What is the risk of modifying the Final Amount formula? How would you mitigate it for a production delivery?
  2. How would you document this extension for the next consultant on the project?
  3. If FintechCo asked for a third adjustment layer six months after go-live, how would you add it? What's the simplest change?
  4. Why do we use an access driver (DCA) on the adjustment line items instead of just hiding them in the UX?
  5. What happens to your formula override if Anaplan releases an IFP 2.1 upgrade?
