🔍 Verifikation nach der Generierung — Workshop vs. Realität

Diese Seite deckt die wesentlichen Hochprioritätsprüfungen für ein funktionierendes Modell ab.

In einer echten Produktivimplementierung ist die Validierung nach der Generierung deutlich umfangreicher — typischerweise 1–2 Tage systematische Überprüfung, die Folgendes abdeckt:

  • Konfigurationsverifikation (Hierarchiestruktur, Dimensionseinstellungen)
  • Validierung von Sicherheit und Zugriffsrechten
  • Performance-Optimierung und Formel-Effizienz
  • Dokumentation und Übergabevorbereitung
  • Validierung des Datenladens (Actuals-Integration)
  • User-Acceptance-Test-Szenarien

Die vollständige Checkliste nach der Generierung finden Sie in Abschnitt 5 und Anhang A des Configuration Guide.

Dieser Workshop konzentriert sich auf Kernkonzepte und Arbeitsablauf. Für Live-Implementierungen folgen Sie stets der vollständigen Configuration Guide-Checkliste.

Erforderlich nach jeder Generierung

Nach Abschluss der Generierung arbeiten Sie die folgende Checkliste durch, bevor Sie eine andere Arbeit ausführen. Dies stellt sicher, dass Sie eine vollständige, stabile Basis haben, bevor Sie Daten oder Erweiterungen hinzufügen.

  • Überprüfen Sie, dass alle 4 Modelle im Workspace vorhanden sind (Financial Planning, Headcount, Admin, CapEx)
  • Prüfen Sie die Anzahl der Applikationsobjekte — Application Framework-Übersicht zeigt: Modelle, UX Apps, Assets, Data Orchestrator-Objekte
  • Öffnen Sie die Error Logs und notieren Sie die Gesamtfehleranzahl — erfassen Sie dies als Ihre Basis
  • Prüfen Sie den Error Log, um zu verifizieren, dass alle Module korrekt generiert wurden — fehlende Module tauchen hier als Fehler auf. Details finden Sie im Abschnitt Error Log Überprüfung.
  • Überprüfen Sie, dass wichtige Line Items in SYS-Modulen befüllt sind: Die Planning Methods-Liste kommt vorausgefüllt. Alle Produktiv-Datenlisten (Entity, Department, Accounts) kommen unbefüllt und werden durch Push aus dem Data Orchestrator befüllt.
  • Bestätigen Sie, dass Hierarchielisten mit den erwarteten Mitgliedern geladen wurden. Alle Mitglieder der Hierarchielisten werden über ADO befüllt — manche über CSV oder eine angebundene externe Quelle. Hierarchien wie Entity, Department, IS Accounts und BS Accounts erfordern einen zusätzlichen Mapping-Schritt im Admin Model, um Quelllisten-Mitglieder ihren Planungs-Äquivalenten zuzuordnen, bevor sie an andere Spoke-Modelle gepusht werden.
  • Prüfen Sie, dass die Mitglieder der Planning Method-Liste befüllt sind — dies ist das einzige Element, das in diesem Stadium verifizierbar ist. Die SYS- und ADMIN-Module, die an die Planning Methods gebunden sind, werden danach befüllt; siehe Anhang A3 des Configuration Guide, um diesen Schritt abzuschließen.
  • Führen Sie eine Stichprobe an einem berechneten Line Item in einem CALC-Modul durch — bestätigen Sie, dass die Formel nicht leer ist
⚠ Leere Formeln? Prüfen Sie zuerst den Error Log

Der wichtigste Weg zu verifizieren, ob eine Formel vorhanden sein sollte und sich als leer herausstellt, ist die Überprüfung des Error Logs. Alle Formula Errors führen dazu, dass die Formel LEER ist — sie schlagen aufgrund einer ungültigen Referenz fehl, entweder durch Line Item-Namensänderungen, die nicht erkannt werden, oder durch Strukturänderungen. Prüfen Sie immer den Error Log, bevor Sie annehmen, dass eine Formel fehlt.

Mini-Übung nach der Generierung

1
Überprüfen Sie Ihr generiertes Modell
⏱ 20 Min.
Ziel Arbeiten Sie die Checkliste nach der Generierung durch. Identifizieren Sie die Basisfehleranzahl und bestätigen Sie, dass die Modellstruktur vollständig ist.
  1. 1Bestätigen Sie aus Ihrem Workspace, dass 4 Modelle aufgelistet sind. Benennen Sie sie in Ihren Notizen.
  2. 2Öffnen Sie die Übersichtsseite des Application Framework. Erfassen Sie: Anzahl der Modelle, Anzahl der UX Apps, Anzahl der Assets, Anzahl der Fehler.
  3. 3Klicken Sie auf Error Logs (unten rechts). Erfassen Sie die Gesamtfehleranzahl — dies ist Ihre Basis.
  4. 4Öffnen Sie das Financial Planning-Modell. Navigieren Sie zu Modules. Überprüfen Sie, dass mindestens ein Modul für jedes Präfix existiert: ADMIN, SYS, DAT, INP, CALC, OUT.
  5. 5Öffnen Sie ein SYS-Modul. Suchen Sie die Planning Methods-Liste. Überprüfen Sie, dass die Methoden, die Sie im Application Framework Configurator aktiviert haben, vorhanden sind.
  6. 6Öffnen Sie ein CALC-Modul (z. B. CALC OPX IS Itemized oder ähnlich). Öffnen Sie ein Line Item im Blueprint. Überprüfen Sie, dass das Formel-Feld nicht leer ist.
  7. 7Markieren Sie Ihre Checkliste als abgeschlossen. Sie sind bereit, die Error Logs zu überprüfen.
