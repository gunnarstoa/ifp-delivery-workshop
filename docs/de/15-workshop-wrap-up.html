Der Trainings-Tenant wird von vielen Gruppen gemeinsam genutzt. Diese fünf Schritte vor Ihrem Verlassen halten den Tenant für die nächste Teilnehmergruppe in einem sauberen Zustand — und sorgen dafür, dass diese Workshops nachhaltig durchführbar bleiben.

⚠ TRAIN-Modelle oder -Apps nicht löschen

Die vorbereiteten TRAIN-Modelle und -Applikationen (jene mit dem Präfix TRAIN, z. B. IFPv2.0.0-train-base-financial-planning) gehören allen Teilnehmern und müssen erhalten bleiben. Löschen Sie nur die Apps, Modelle und Objekte, die Sie während der Übungen erstellt haben.

  • Löschen Sie die Applikationen, die Sie während der Übungen konfiguriert haben — jene ohne das Präfix TRAIN
  • Löschen Sie die Modelle, die aus Ihren konfigurierten Applikationen generiert wurden — jene ohne das Präfix TRAIN
  • Löschen Sie alle Objekte, die Sie während der Übungen Ihrem zugewiesenen Dataspace hinzugefügt haben
  • Setzen Sie Hierarchien und Konfigurationsfragen im Application Framework Configurator auf die Standardwerte zurück (eine Hierarchie nach der anderen — es gibt kein globales Reset)
  • Stellen Sie das Financial Planning-Modell auf seine ID vor der Erweiterung zurück
    • Für Application Training-Tenants 00 bis 60 lautet die ID vor der Erweiterung 2064565
    • Für Tenants, die in EMEA-Workshops verwendet werden, kann die ID abweichen — klären Sie dies vor dem Zurücksetzen mit dem EMEA-Tenant-Admin ab
