Erweiterungsübung 2: Channel Revenue Planning

Eine neue flache Liste (Channel) und ein Revenue-Planning-Modul anlegen, integriert in Gross Profit Planning

Erweiterungen60 Min.Angeleitete Übung
🌐 Hinweis zur Übersetzung

Dies ist eine automatische Übersetzung der englischen Originalfassung. Die maßgebliche und aktuellste Version finden Sie hier auf Englisch.

Geschäftsanforderung

Es wird eine Art der Umsatzplanung benötigt, beispielsweise nach Channel.

Lösung

  • Neue flache Channel-Liste mit Top Level „All Channels" hinzufügen
  • Ein neues Modul anlegen, das die bestehenden Umsatzplanungsdimensionen plus die zusätzliche Channel-Dimension nutzt
  • Neue Line Items zum betroffenen Modul hinzufügen
  • Die Aggregationsformel so anpassen, dass die neuen Line Items einbezogen werden

Schritte

Schritt 1: Die Channel-Liste anlegen

  1. Legen Sie im Data Model Editor eine neue Liste: EXT Channel an
    • Namenskonvention: Das Präfix „EXT" zeigt an, dass es sich um eine Erweiterung handelt (nicht automatisch generiert)
    • Flache Liste (keine Hierarchie) — einfach diskrete Channel-Namen
  2. Fügen Sie das Top Level-Element hinzu: „All Channels"
    • Dieses aggregiert alle einzelnen Channel-Werte
  3. Fügen Sie 3 Beispiel-Channels hinzu:
    • Channel 1
    • Channel 2
    • Channel 3
  4. Liste speichern
ℹ Namenskonvention: EXT-Präfix

Als Erweiterungen angelegte Listen und Module erhalten das Präfix „EXT", um sie von automatisch generiertem Inhalt zu unterscheiden. Das erleichtert es, bei Upgrades nachzuvollziehen, was nach der Generierung hinzugefügt wurde.

Schritt 2: Das Channel-Revenue-Planning-Modul anlegen

  1. Legen Sie ein neues Modul an: EXT INP Channel Revenue Planning
  2. Definieren Sie die Modul-Dimensionen:
    • Time (Standarddimension)
    • Alle Dimensionen aus INP Gross Profit Planning (Product, Job, Entity usw.)
    • Channel (Ihre neue Dimension — Liste EXT Channel)
  3. Fügen Sie dem Modul Line Items hinzu:
    • Price — Stückpreis pro Channel
    • Volume — verkaufte Mengen pro Channel
    • Total — berechneter Umsatz (Price × Volume)
  4. Konfigurieren Sie das Line Item „Total" mit der Formel: Price × Volume
  5. Modul speichern

Schritt 3: Eine neue Planungsmethode hinzufügen

  1. Öffnen Sie das Modul „SYS by Planning Methods"
  2. Fügen Sie in den „List Planning Methods" einen neuen Eintrag hinzu: „Channel Planning"
  3. Änderung speichern
⚠ Planungsmethoden müssen registriert werden

Jeder neue Planungsansatz (Channel Planning, Direct Input usw.) muss in der Liste Planning Methods registriert werden. Ohne diese Registrierung erkennt die UX ihn nicht als gültige Option für Benutzer.

Schritt 4: Verfügbarkeit der Planungsmethode konfigurieren

  1. Öffnen Sie das Modul „ADMIN Define Product to Planning Method"
  2. Fügen Sie für „Service Offering 1" (oder Ihr Zielprodukt) „Channel Planning" zur Liste der verfügbaren Planungsmethoden hinzu
  3. Damit teilen Sie dem System mit: „Service Offering 1 kann mittels Channel Planning geplant werden"
  4. Änderungen speichern

Schritt 5: Das Gross-Profit-Planning-Modul aktualisieren

Integrieren Sie nun Channel Planning in den Haupt-Planungsfluss, indem Sie Referenz-Line Items zu INP Gross Profit Planning hinzufügen:

  1. Öffnen Sie das Modul „INP Gross Profit Planning"
  2. Fügen Sie 3 neue Line Items vor dem Line Item „Final Amount" hinzu:
    • Channel Planning — Überschrift/Abschnittsindikator
      • Als Heading 1 setzen (nur Styling/Organisation, keine Daten)
    • Use Channel Planning Method? — Boolean-Flag
      • Format: Boolean
      • Zweck: Der Benutzer wählt „Yes", um die Planung auf Channel-Ebene zu aktivieren
      • Einstellungen von einem ähnlichen Line Item übernehmen (z. B. „Use COGS Pct of Revenue Method?")
    • Channel Planning Revenue — das aggregierte Ergebnis
      • Format: gleich wie „Final Amount"
      • Dieses Line Item zieht Daten aus EXT INP Channel Revenue Planning und summiert nach Channel

Schritt 6: Die Formel für Channel Planning Revenue erstellen

  1. Erstellen Sie für das Line Item „Channel Planning Revenue" eine Formel, die:
    • Auf EXT INP Channel Revenue Planning, Line Item „Total" verweist
    • Über die Channel-Dimension summiert (alle Channels aggregiert)
    • Nur dann berechnet, wenn „Use Channel Planning Method?" = True
    • 0 oder leer zurückgibt, wenn die Methode nicht gewählt ist
  2. Formel-Muster (Beispiel):
    • IF(Use Channel Planning Method?, SUM(EXT INP Channel Revenue Planning.Total), 0)
  3. Passen Sie die genaue Formel an die Benennung und Struktur Ihres Modells an
  4. Dokumentieren Sie die Formel im Notes-Feld des Modul-Blueprints

Schritt 7: Line Item Subsets (LISS) aktualisieren

Registrieren Sie Ihre neuen Line Items in den passenden Subsets, damit sie auf den richtigen UX-Seiten erscheinen:

  1. Öffnen Sie „LISS: Margin Filter"
  2. Setzen Sie die Boolean-Markierung für Ihre 3 neuen Line Items:
    • Channel Planning (Überschrift)
    • Use Channel Planning Method?
    • Channel Planning Revenue
  3. Änderungen speichern

Schritt 8: Aggregationsformeln aktualisieren

Aktualisieren Sie die Formeln in bestehenden Line Items, sodass Channel Planning Revenue einbezogen wird:

  1. Öffnen Sie im Modul „INP Gross Profit Planning" die Formel für „Final Amount"
  2. Aktualisieren Sie die Formel und ergänzen Sie:
    • + Channel Planning Revenue (zusätzlich zu den bestehenden Bestandteilen)
  3. Aktualisieren Sie die Formel für „SUM: Revenue", falls vorhanden:
    • Ergänzen Sie + Channel Planning Revenue, damit channelbasierter Umsatz in den Summen enthalten ist
  4. Dokumentieren Sie beide Formeländerungen in Ihrem Offline-Änderungsprotokoll
ℹ Strategie für Formel-Updates

Wenn Sie bestehende Formeln aktualisieren, um auf neue Erweiterungen zu verweisen, nutzen Sie bedingte Logik:

  • IF(Use Channel Planning Method?, Channel Planning Revenue, 0)

Das verhindert Fehler, wenn die Methode nicht gewählt ist, und macht die Logik für künftige Wartende verständlich.

Schritt 9: In der UX validieren

  1. Öffnen Sie die UX-App
  2. Navigieren Sie zur Gross-Profit-Planning-Seite für ein Beispielprodukt (Service Offering 1)
  3. Verifizieren Sie:
    • Der „Use Channel Planning Method?"-Boolean erscheint und ist editierbar
    • Wenn Sie ihn auf „Yes" setzen, passt sich die Planungsoberfläche an
    • Sie können zur Channel-Revenue-Planning-Seite navigieren und Price und Volume eingeben
    • Die Summen fließen korrekt zurück in Gross Profit Planning
    • Alle Summenberechnungen sind korrekt
  4. Wenn etwas fehlt oder defekt ist:
    • Prüfen Sie, dass alle Line Items dem LISS hinzugefügt wurden
    • Verifizieren Sie, dass die Formel-Syntax zu den Namenskonventionen Ihres Modells passt
    • Vergleichen Sie mit einem funktionierenden Referenzmodul

Wichtige Erkenntnisse

  • Erweiterungen erhalten die generierte Struktur — Sie fügen hinzu, ersetzen nicht
  • Planungsmethoden organisieren Benutzerentscheidungen — registrieren Sie neue Planungsansätze immer
  • Line Item Subsets steuern die Sichtbarkeit — neue Inhalte erscheinen nur, wenn sie dem LISS hinzugefügt werden
  • Formeln müssen bedingt sein — nur berechnen, was der Benutzer ausgewählt hat
  • Dokumentation ist entscheidend — jede Änderung muss für künftige Upgrades protokolliert werden
  • End-to-End testen — von der Dateneingabe bis zur finalen Summenberechnung

⚠ Upgrade-Überlegung

Wenn Anaplan eine neue IFP-Version veröffentlicht, müssen alle Erweiterungen, die Sie gebaut haben, auf das neu generierte Modell erneut angewendet werden. Detaillierte Dokumentation (was Sie geändert haben, warum, originale vs. neue Formeln) macht diesen Prozess deutlich schneller und weniger fehleranfällig.
