Was die Generierung bewirkt

Wenn Sie die Generierung starten, liest das Application Framework alle 44 Ihrer Konfigurationsantworten und provisioniert die vollständige IFP-Modellstruktur. Dies ist keine Template-Kopie — es ist ein dynamischer Generierungsprozess, der Ihre Konfigurationsentscheidungen anwendet, um ein für Ihre Antworten spezifisches Modell zu erzeugen.

Die Generierung provisioniert vier Modelle:

  • Financial Planning — das Hauptplanungsmodell (INP-, CALC-, OUT-Module)
  • Headcount — das HC-Modell mit Planung auf Job-Ebene (sofern aktiviert)
  • Admin — Hauptort für Mappings von Quell- zu Planungsmetadaten für Kontenhierarchien, Entity und Department.
  • CapEx — CapEx-Planungsmodell (nur sofern aktiviert)

🚨 „Generated with Errors" bedeutet nicht defekt

„Generated with Errors" ist normal und zu erwarten, besonders im Training und bei frühen Implementierungen. Es bedeutet, dass einige Formeln Elemente referenzieren, die sich vom Basismodell-Template unterscheiden — üblicherweise verursacht durch Umbenennen von Hierarchien, das Entfernen von Geografien oder Konfigurationsentscheidungen, die von den Template-Annahmen abweichen. Das Modell ist funktionsfähig. Sie arbeiten die Fehlerliste systematisch im Abschnitt Error Log Überprüfung durch.

Generierungsschritte

1

Generierung starten

Klicken Sie im Application Framework Configurator nach Abschluss aller Abschnitte auf Generate. Bestätigen Sie alle Dialoge. Notieren Sie die Startzeit.

2

Status überwachen

Die Application Framework-Übersichtsseite zeigt den Generierungsstatus. Aktualisieren Sie regelmäßig. Schließen Sie nicht den Browser und starten Sie während des laufenden Vorgangs keine weitere Generierung.

3

Erster Blick auf das generierte Modell

Sobald die Generierung abgeschlossen ist, navigieren Sie zu Ihrem Workspace. Sie sollten die vier IFP-Modelle aufgelistet sehen. Öffnen Sie das Financial Planning-Modell und überprüfen Sie, dass die Planungs-Subprozesse vorhanden sind — einschließlich Revenue & COGS, Operating Expense, Headcount, CapEx Planning und Balance Sheet & Cash Flow (abhängig von Ihrer Konfiguration).

4

Error Logs prüfen

Kehren Sie zum Application Framework zurück. Unten rechts auf der Übersichtsseite finden Sie die Anzahl der Error Logs. Klicken Sie, um das Log zu erweitern. Hinweis: Mehr als 0 Fehler ist normal. Fahren Sie mit Error Log Überprüfung fort.
